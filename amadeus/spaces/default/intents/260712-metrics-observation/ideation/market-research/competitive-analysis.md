# 競合分析 — コード品質メトリクスの時系列観測(260712-metrics-observation)

> 対象問題: 「回帰防止ゲートはあるが時系列の観測機構がない」(intent-statement.md)。本書は既存ソリューションの能力を本リポジトリの前提(Bun 単独ランタイム・ランタイム依存追加の禁止 project.md Forbidden・GitHub Actions CI・Codecov 導入済み)に照らして比較する。市場の一般的能力に関する記述は各製品の公知の中核機能に基づく(確信度: 高)が、価格・制限の細部は変動するため実装時に一次確認する(仮説マーク ※)。

## 比較対象

| ソリューション | 時系列能力 | 本 repo 前提との適合 | 主な不適合 |
|---|---|---|---|
| **Codecov(導入済み)** | カバレッジ%の履歴・トレンドグラフを SaaS 側が自動保持 | ◎ 既に全 PR/main で計測・アップロード済み(coverage:ci)。追加コストゼロ | カバレッジ以外(CCN 分布・LOC・テスト数・dist サイズ)は対象外。履歴データは SaaS 側にありリポジトリ内に残らない(オフライン参照・独自集計に難) |
| **SonarQube / SonarCloud** | 複雑度・重複・カバレッジ等の履歴を DB 保持、ダッシュボード | △ SaaS(SonarCloud)なら CI 統合は容易 | スキャナ実行の CI 負荷+外部サービス依存の新設。TypeScript の CCN 計測が lizard と異なる定義になり、既存 complexity-gate(lizard 正)と**二重の物差し**を作る。self-host 版はサーバ運用が必要で今のチーム態勢に不釣り合い |
| **CodeClimate / CodeScene** | 品質トレンド・ホットスポット分析 | △ SaaS 統合自体は可能 | 同上(外部依存・物差し二重化)。CodeScene の挙動解析は本件の「定点観測」ニーズに対して過剰 ※ |
| **github-action-benchmark 等の履歴コミット型 Action** | 計測値を gh-pages / ブランチへ追記しグラフ化 | ○ リポジトリ内に履歴が残る発想は本件と同型 | ベンチマーク(性能)向けの様式で、任意メトリクスの台帳としては変換層が必要。Action 依存の追加 |
| **自作 snapshot(既存ツール再利用)** | metrics/ 配下へのコミット済み JSON(日付付き or 追記台帳) | ◎ lizard 計測は complexity-gate.ts が既に内蔵・カバレッジは coverage:ci の lcov が既にある・LOC/ファイル数/テスト数は git/静的走査で決定的に取れる。Bun 単独・依存追加ゼロで成立 | 可視化(グラフ)は別途(ただし Issue #921 は「ファイルに出力し定点観測」が主要件で、可視化は論点扱い) |

## 本リポジトリ固有の適合性所見(実測ベース)

- **lizard 計測ロジックの再利用可能性**: `tests/complexity-gate.ts` が lizard 実行・パース・関数別 CCN の取得までを既に実装(#837)。時系列 snapshot はこの出力の「保存」を足すだけで、計測の物差しがゲートと**同一**に保たれる(二重物差し回避は competitive 上の決定的な差別化点)。
- **カバレッジ履歴**: Codecov が SaaS 側で既に保持(過去 PR で pulls API から取得実績あり — batch7 の waiver 手続きで実証済み)。ただし「リポジトリ内の台帳」という #921 の要望形とは異なるため、snapshot に lcov 集計値を含めるかは requirements の論点。
- **追記型台帳のリスク**: 複数 PR が同一ファイルへ追記すると textual conflict(cid:shared-ledger-insert-collision の実測クラス)— 日付付き個別ファイル方式が構造的に安全。この知見は本チームの実障害由来で、外部製品比較では得られない設計入力。

## 結論(市場ポジション)

外部 SaaS は「可視化つき時系列」の完成品だが、(a) 物差しの二重化 (b) 外部依存の新設 (c) リポジトリ内台帳という要望形との不一致、の3点で本件の適合度が低い。**既存ツール(lizard 経路・lcov・git 統計)の出力を保存する薄い自作 snapshot が、依存ゼロ・物差し統一・要望形一致の3点で優位**。カバレッジのトレンド可視化だけは Codecov(導入済み)がすでに提供しており、重複構築しない。
