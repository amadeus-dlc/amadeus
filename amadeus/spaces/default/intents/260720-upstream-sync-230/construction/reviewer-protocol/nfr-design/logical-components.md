# Logical Components — reviewer-protocol

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存orchestrator、reviewer、package、test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| Orchestrator Scope Builder | `reviewerReadScope`でcurrent Unitのclosed pass-listを構築 | UnitRef + consumes→ReadScope | invocation単位 |
| Internal Artifact Enumerator | stage/Q&A/required/実在optional/passed consumesを列挙 | directive paths→deduplicated paths | current Unit単位 |
| Internal Owner Resolver | current Unitとpassed contractから単一ownerを解決 | integration ID→0/1/複数owner | scope decision単位 |
| Internal Decision Recorder | 4条件をread前評価し同値証跡を作る | path/reason/ID/evidence→approved/rejected | invocation単位 |
| Reviewer Identity Builder | `runtimeReviewIdentity`で4-field Reviewを生成 | checker persona + UTC→ReviewHeader | review iteration単位 |
| Internal UTC Runner | 単一`date -u`を直前実行し検証 | process result→validated UTC | Review書込単位 |
| Existing Reviewer Subagent | passed artifactsを独立検査しprimary artifactへ追記 | explicit paths→READY/NOT-READY Review | Unit review単位 |
| Existing Review/Audit Records | scope decisionとmaker-checkerを追跡 | subagent result + Review→既存audit evidence | event shard単位 |
| Package Projector | authored contractを6 harnessへ生成投影 | source/manifest→generated bytes | package generation単位 |
| Self-install Projector | closed list 4面をpromote/check | generated source→install surfaces | self-install面単位 |
| Verification Fixtures | pass-list、identity、scope、projectionを対照検査 | positive/negative fixtures→evidence | test process単位 |

## Public・internal boundary

公開面は次の正準2関数だけである。

```ts
function reviewerReadScope(unit: UnitRef, consumes: readonly ArtifactRef[]): ReadScope;
function runtimeReviewIdentity(persona: ReviewerPersona, utcDate: string): ReviewHeader;
```

artifact enumerator、owner resolver、decision recorder、date runnerは内部helperであり、追加public APIではない。scope builderはorchestrator、identity builderは実checkerのreview実行境界が所有する。

## Interaction sequence

1. engine directiveをOrchestrator Scope Builderへ渡し、current Unitのstage、Q&A、実在成果物、passed consumesからclosed listを作る。
2. spot-checkが必要ならOwner ResolverとDecision Recorderがcurrent Unitとpassed contractだけで4条件をread前評価する。
3. approved時だけ単一owner pathを当該invocationへ加え、Reviewer Subagentへ明示pathを渡す。rejected時は追加read 0でfindingへ閉じる。
4. reviewerはvalidationをpass-list内へ限定し、Review直前にUTC Runnerを一度呼ぶ。
5. Identity Builderが実checker personaとvalidated UTCから4-field Reviewを作り、primary artifactへ追記する。
6. Package/Self-install ProjectorとVerification Fixturesが6/4面、scope、identity、driftを全数検査する。

## Isolation・shared resources

shared resourceはversion-controlled authored source、passed contract、既存audit、package manifestだけである。各scope decisionとReviewはinvocation-localで、cache、queue、network、database、service、AWS infrastructure、別provenance storeを共有しない。blast radiusはUnit、review iteration、projection gateへ限定する。
