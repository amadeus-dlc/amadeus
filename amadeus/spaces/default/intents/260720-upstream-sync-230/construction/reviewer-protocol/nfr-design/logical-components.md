# Logical Components — reviewer-protocol

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存orchestrator、reviewer、package、test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| Orchestrator Scope Builder | `reviewerReadScope`でcurrent Unitのclosed pass-listを構築 | UnitRef + consumes→ReadScope | invocation単位 |
| Internal Artifact Enumerator | `stage_file`/required `produces`/実在optional `produces`/present consumesを列挙。Q&Aはconsumes明示時だけ | directive paths→deduplicated paths | current Unit単位 |
| Internal Owner Resolver | current Unitとpassed contractから単一ownerを解決 | integration ID→0/1/複数owner | scope decision単位 |
| Internal Decision Recorder | `check-read`で4条件をrequest前評価しdeclared transcriptを作る | path/reason/ID/evidence→approved/rejected | invocation単位 |
| Internal Transcript Validator | directive/artifacts/consumesからScope decision transcriptを全件再計算する | prompt/result transcript→accepted/rejected | Review受理単位 |
| Reviewer Identity Builder | `runtimeReviewIdentity`で4-field Reviewを生成 | checker persona + UTC→ReviewHeader | review iteration単位 |
| Internal UTC Runner | 単一`date -u`を直前実行し検証 | process result→validated UTC | Review書込単位 |
| Existing Reviewer Subagent | passed artifactsを独立検査しprimary artifactへ追記 | explicit paths→READY/NOT-READY Review | Unit review単位 |
| Existing Review/Audit Records | scope decisionはprompt/result/Review、maker-checker identity/completionは既存auditで追跡 | subagent result + Review→既存audit evidence | event shard単位 |
| Package Projector | authored contractを6 harnessへ生成投影 | source/manifest→generated bytes | package generation単位 |
| Self-install Projector | closed list 4面をpromote/check | generated source→install surfaces | self-install面単位 |
| Verification Fixtures | pass-list、identity、scope、projectionを対照検査 | positive/negative fixtures→evidence | test process単位 |

## Public・internal boundary

公開面は次の正準2関数だけである。

```ts
function reviewerReadScope(unit: UnitRef, consumes: readonly ArtifactRef[]): ReadScope;
function runtimeReviewIdentity(persona: ReviewerPersona, utcDate: string): ReviewHeader;
```

artifact enumerator、owner resolver、decision recorder、transcript validator、date runnerは内部helperであり、追加public APIではない。scope builderはorchestrator、identity builderは実checkerのreview実行境界が所有する。

## Interaction sequence

1. engine directiveをOrchestrator Scope Builderへ渡し、`stage_file`、current Unitの実在`produces`、present consumesからclosed listを作る。Q&Aは`directive.consumes`明示時だけ含める。
2. spot-checkが必要ならOwner ResolverとDecision Recorderがcurrent Unitとpassed contractだけで4条件を`check-read`によりrequest前評価する。
3. approved時だけ単一owner pathを当該invocationへ加え、Reviewer Subagentへ明示pathとtransient transcriptを渡す。rejected時はaccepted request 0でfindingへ閉じる。
4. Transcript Validatorがresultの全entryを同じdirective/artifacts/consumesから再計算し、bypass、改竄、rejected/outside/2 file目requestを拒否する。
5. reviewerはvalidationをdeclared pass-list内へ限定し、Review直前にUTC Runnerを一度呼ぶ。
6. Identity Builderが実checker personaとvalidated UTCから4-field Reviewを作り、primary artifactへ追記する。
7. Package/Self-install ProjectorとVerification Fixturesが6/4面、scope、identity、driftを全数検査する。

## Isolation・shared resources

shared resourceはversion-controlled authored source、passed contract、既存audit、package manifestだけである。Scope decision transcriptはprompt/result間だけを通るtransient carrierで、最終Reviewには再検証済みScope decision projectionをprimary artifactの一部として永続記録する。cache、queue、network、database、service、AWS infrastructure、別provenance store、actual-read ledger、proxy/sandboxを共有しない。blast radiusはUnit、review iteration、projection gateへ限定する。
