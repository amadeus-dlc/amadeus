# Logical Components — routing-and-autonomy-guards

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存CLI、hook、doctor、transaction、package/test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| Help Intent Classifier | `classifyHelpIntent`の共有decision table | tokens→help/workspace/freeform | invocation単位 |
| Engine Routing Adapter | classifierをorchestrate parserへ投影 | argv→engine route | next invocation |
| Terminal Routing Adapter | classifierをpre-LLM terminal判断へ投影 | argv→terminal/freeform | prompt invocation |
| Utility Namespace Guard | intent/space help backstopとreserved slug拒否 | verb/slug→result/error | namespace mutation前 |
| Compose Marker Inspector | `inspectComposeMarker`で単一statを分類 | stat/now/TTL→marker result | workspace marker単位 |
| Stop Hook Adapter | autonomy、carve-out、janitorを順序制御 | state+marker→decision/outcome | Stop invocation |
| Doctor Adapter | 同一marker seamをread-only投影 | marker result→doctor row | doctor probe |
| Recompose Guard | `assertRecomposeAllowed`でmutation前拒否 | state snapshot→allowed/denied | transaction単位 |
| Existing Audit-lock Transaction | allowed recomposeのstrict validation/write | snapshot→atomic mutation | active intent |
| Package Projector | authored sourceを6 harnessへ生成 | source/manifest→generated bytes | package gate |
| Self-install Projector | closed list 4面をpromote/check | generated source→install面 | self-install gate |
| Verification Fixtures | routing、marker、atomicity、projectionを対照検査 | fixtures→evidence | test process |

## Public・internal boundary

公開面は次の正準3 seamだけである。

```ts
function classifyHelpIntent(tokens: readonly string[]): HelpRouting;
function inspectComposeMarker(observation: MarkerObservation, nowMs: number, ttlMs: number): MarkerFreshness;
function assertRecomposeAllowed(autonomy: ConstructionAutonomy): RecomposeGuardResult;
```

reserved slug guard、path derivation、stat/unlink adapter、doctor renderer、remediation formatterは内部helperであり、追加public APIではない。consumerとfixtureを同じBoltで着地させ、dormant helperを先行導入しない。

## Interaction sequence

1. Engine/Terminal/Utility adaptersがHelp Intent Classifierを共有し、namespace guardをmutation前に適用する。
2. Stop Hook Adapterはautonomyを先に判定し、non-autonomous時だけMarker Inspectorを呼ぶ。freshだけallow-stop、staleはdecision確定後に単一unlinkを試す。
3. Doctor Adapterは同じMarker Inspectorをread-onlyで呼び、path/bytes/mtimeを変更しない。
4. Recompose Guardはlock内state snapshot直後に評価され、deniedならtransaction consumerへ渡さない。
5. ProjectorとFixturesが3入口、marker経路、mutation 0、6/4面を全数検査する。

## Isolation・shared resources

shared resourceはworkspace-level単一marker、active intent state/audit、authored source、package manifestだけである。各decisionはinvocation-localで、daemon、background janitor、cache、queue、network、database、service、AWS infrastructureを共有しない。blast radiusはrouting invocation、marker、transaction、projection gateへ限定する。
