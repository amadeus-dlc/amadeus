# Logical Components — harness-hook-correctness

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存C6 adapter/package/test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| Authored Harness Adapters | 実在hook spawn siteとhost payload受領 | host event→canonical seam input | harness adapter単位 |
| Runtime Spawn Seam | `spawnHookWithRuntime(args, input)`で絶対runtime起動 | args/HookInput→SpawnResult | child invocation単位 |
| Kiro Context Seam | `parseKiroIdeHookContext(payload)`で判別union化 | unknown→HookContextResult | Kiro IDE event単位 |
| Claude Command Seam | `renderClaudeHookCommand(variable, hook)`で11 command引用 | variable/HookSpec→string | command render単位 |
| Internal Classifiers | event、path、identity、drop、settings traversal | public seam内部のvalidated values | public result内に封じる |
| Core Audit/Sensor Targets | canonical contextをaudit-firstで消費 | canonical payload→既存audit/advisory | target invocation単位 |
| Runtime/State Targets | payload-free audit-tail self-gate、forward-only sync | audit tail→既存guard結果 | current intent state guard内 |
| Package Projector | authored sourceから6 harness projectionを生成 | manifest/source→generated bytes | package generation単位 |
| Self-install Projector | closed list 4面をpromote/check | generated source→4 install面 | self-install面単位 |
| Verification Fixtures | unit/integration/e2eとbyte/drift検査 | fixtures→test evidence | test process単位 |

## Public・internal boundary

公開面は次の3関数だけである。

```ts
function spawnHookWithRuntime(args: readonly string[], input: HookInput): SpawnResult;
function parseKiroIdeHookContext(payload: unknown): HookContextResult;
function renderClaudeHookCommand(projectDirVariable: "$CLAUDE_PROJECT_DIR", hook: HookSpec): string;
```

event classifier、path/identity抽出、visible drop記録、settings traversalは各public seam内部helperであり、別component APIとして公開しない。host固有payloadをcore型へ昇格せず、adapterがcanonical resultへ翻訳する。

## Interaction sequence

1. Authored adapterがhost inputを受領し、該当public seamへ渡す。
2. Runtime flowはabsolute Bun argvでchild core hookを起動し、既存process resultを返す。
3. Kiro flowはcontext resultを分類し、成功writeだけをaudit-first→sensorへ渡す。payload-free targetはaudit tailを別入力として既存guardへ渡す。
4. Claude flowはauthored settingsの11 hookだけをrenderし、package projectorが6面へ導出する。
5. Verification fixturesがsource/projection、6/4面、11 command、対象外bytesを対照する。

## Isolation・shared resources

shared resourceはversion-controlled authored source、既存audit/state、package manifestだけである。各public seamはinvocation-localで、cache、queue、database、network、AWS infrastructure、daemonを共有しない。blast radiusはadapter/event/command/projection gateへ限定され、新service isolationやdeployment unitを追加しない。
