# Logical Components — stage-contract

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存schema、graph、directive、coverage、approval、package/test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| UnitKind Vocabulary | closed 5語彙の単一所有 | raw kind→known kind/error | Unit edge |
| Stage Frontmatter Validator | exact-shapeとmap整合を検証 | raw→typed stage/error | stage file |
| Internal Parser/Emitter | source表現をtyped shapeへ往復 | bytes↔typed value | parse/emit invocation |
| Stage Graph Compiler | validated stagesをtyped graphへ投影 | stages→GraphStage[] | graph compile |
| Internal Kind Filter | required+optional unionを一回filter | candidates+kind→applicable set | stage/Unit pair |
| Directive Consumer | applicable候補をengine directiveへ投影 | snapshot→produces | per-unit invocation |
| Coverage Consumer | applicable required subsetを検査 | files→covered/uncovered | stage/Unit pair |
| Approval Guard | 全Unit coverageを同snapshotで判定 | coverage→allow/deny | stage gate |
| Package Projector | authored sourceを6 harnessへ生成 | source/manifest→projection | package gate |
| Self-install Projector | closed list 4面をpromote/check | generated source→install面 | self-install gate |
| Verification Fixtures | schema/bytes/pruning/approvalを対照検査 | fixtures→evidence | test process |

## Public・internal boundary

公開面は次の正準4 seamだけである。

```ts
function validateStageFrontmatter(raw: unknown, context: ValidationContext): ContractResult<StageFrontmatter>;
function normalizeUnitKind(raw: unknown): ContractResult<UnitKind>;
function compileStageGraph(stages: readonly StageFrontmatter[]): ContractResult<readonly GraphStage[]>;
function requiredArtifactsForUnit(stage: GraphStage, kind: UnitKind): readonly string[];
```

`parseStageFrontmatter`、`emitStageFrontmatter`、`filterProducesByKind`は内部helperであり、追加public APIではない。kind未指定時はsignatureを広げずcallerがfull matrixへ委譲する。

## Interaction sequence

1. Parser/Validatorがraw frontmatterを検証済みstageへ変換する。
2. Compilerが単一UnitKind vocabularyとvalidated stageからtyped graph snapshotを作る。
3. Kind Filterがrequired+optional unionを一回pruneし、DirectiveとCoverageへ同じresultを渡す。
4. Coverageはrequired subsetだけを検査し、Approval Guardが全Unitを同snapshotで判定する。
5. ProjectorとFixturesが5/12/6/4 matrix、bytes、failure、vacuous境界を検査する。

## Isolation・shared resources

shared resourceはauthored stage source、lib vocabulary、typed graph、package manifestだけである。pure validation/filterはstate/audit/generated bytesを変更せず、new service、database、network、schema registry、when evaluatorを持たない。

## トレーサビリティ

inventoryは`business-logic-model.md`のPublic seam/Contract pipeline/Coverage、`performance-requirements.md`のsingle-pass、`security-requirements.md`のtrust boundary、`scalability-requirements.md`の5/12/6/4 closed set、`reliability-requirements.md`のcorrectness invariants、`tech-stack-decisions.md`のsource/test ownershipへ対応する。
