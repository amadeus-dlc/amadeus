# Security Design — stage-contract

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。raw frontmatter、Unit edge、runtime graphをuntrusted inputとしてfail-closedに検証する。

## Trust boundaries

| Boundary | Control | Rejected behavior |
|---|---|---|
| raw frontmatter→typed stage | exact-shape/known field validation | wrong type、empty、malformed、unknown predicate |
| raw kind→UnitKind | lib closed 5語彙 | unknown kind、第二registry |
| produces_kinds→artifact map | key/value/reference/duplicate validation | orphan、empty list、unknown kind |
| Unit edge→runtime graph | name順序/type/kind validation | kind-before-name、silent ignore |
| typed graph→consumers | single snapshot/filter | directive/coverage非対称、speculative prune |

## Exact-shape controls

`number`はstringかつ既決pattern、`name`/`bundle`は非空string、`required_sections`は非空string配列、`when`はexactly-one-keyの既決producer predicateだけを受理する。block/inline whenは同じtyped objectへparseするが評価せず、permissionや実行条件へ昇格しない。

追加field absent時はproperty非生成、valid valueは追加canonicalizationなしで保持する。unknown/wrong/empty/malformedはgraph/state/audit/source/generated bytesを変更する前にfield、Unit、実値、許容値付きerrorで拒否する。

## Supply-chain controls

authored sourceを正本にgeneratorで6 harnessへ投影し、dist手編集を受理しない。新credential、permission、runtime dependency、network、database、service、UI、event、retentionを追加しない。

## トレーサビリティ

本設計は`security-requirements.md`のSEC-U01-01〜04、`performance-requirements.md`のsingle pipeline、`scalability-requirements.md`のsingle vocabulary、`reliability-requirements.md`のbytes不変、`tech-stack-decisions.md`のsource boundary、`business-logic-model.md`のエラーと副作用へ対応する。
