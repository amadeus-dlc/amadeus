# Logical Components — workspace-inspection

> 入力: `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。既存workspace detector、projectors、package/test境界だけをcomponent inventoryとして明示する。

## Component inventory

| Component | Responsibility | Input/Output | Failure domain |
|---|---|---|---|
| Workspace Inspector | `inspectWorkspace`でroot-first orchestration | root+I/O→WorkspaceScanResult | scan invocation |
| Root Signal Evaluator | language/framework/build/source/manifest判定 | directory→signals/advisory | root/candidate |
| Depth-One Detector | `detectDepthOneProjects`でcanonical candidates | entries→nested hits | fallback scan |
| Submodule Inspector | `inspectSubmodules`でsafe metadata観測 | gitmodules+tree→submodules/advisory | workspace root |
| Result Classifier | `classified | inconclusive`を確定 | observations→typed union | scan snapshot |
| Birth/State Projector | classifiedだけをcommitへ投影 | result→state fields/error | intent birth |
| Detect Projector | 両variantをread-only JSON化 | result→JSON | detect invocation |
| Doctor Projector | submodule/advisoryをread-only表示 | result→doctor rows | doctor invocation |
| Audit Projector | snapshotからpayloadをpure生成 | result→existing event payload | classified birth |
| Package/Self-install Projector | sourceを6/4面へ投影 | manifest→generated bytes | distribution gate |
| Verification Fixtures | scan/failure/projection/bytesを対照検査 | temp trees→evidence | test process |

## Public・internal boundary

公開面は次の正準3 seamだけである。

```ts
function inspectWorkspace(root: WorkspaceRoot, io: ReadOnlyFs): WorkspaceScanResult;
function detectDepthOneProjects(root: WorkspaceRoot, io: ReadOnlyFs): readonly NestedProject[];
function inspectSubmodules(root: WorkspaceRoot, io: ReadOnlyFs): SubmoduleInspection;
```

path parser、signal evaluator、candidate filter/sorter、aggregation、formatterは内部helperであり、新public APIではない。consumerは二値projectTypeへ早期collapseせず、typed unionをexhaustiveに扱う。

## Interaction sequence

1. Inspectorがroot signalとsubmoduleを一回評価する。Submodule Inspectorがunsafe/parse不能を返した場合は他signalより先にinconclusiveを確定し、safe+unsafe/root+unsafeでもcommit pathへ進まない。全入力がcompleteでsignalがあればfallbackを止める。
2. root無信号時だけDepth-One Detectorがsafe candidatesを同じsignal evaluatorで検査する。
3. Classifierが全観測完了時だけclassified、読取不完全時はinconclusiveを返す。
4. 4 projectorsが同一immutable snapshotをpure projectionし、birthはinconclusiveをmutation前に拒否する。
5. Projector/Fixturesが1/5/4/6/4 matrix、safe path、bytes互換、read-onlyを検証する。

## Isolation・shared resources

shared resourceはread-only workspace tree、invocation-local snapshot、package manifestだけである。submodule init/repair、filesystem index、cache、network、database、service、daemonを持たず、blast radiusをscan/projector invocationへ限定する。
