# Business Rules — plugin-projection

> 上流入力(consumes全数): `unit-of-work.md`、`unit-of-work-story-map.md`、`requirements.md`、`components.md`、`component-methods.md`、`services.md`。

## Discovery and validation rules

| ID | Rule | Failure behavior |
|---|---|---|
| BR-U09-01 | source discoveryはrepository rootの`plugins/<name>/`だけを対象とする。 | 他のtreeやgenerated distをsourceにしない |
| BR-U09-02 | plugin identity、manifest、artifact pathはC1 contractで全件validateしてからbuildする。 | invalid inputをloud failure、write 0で拒否 |
| BR-U09-03 | plugin nameとartifact relative pathはcanonical sortし、filesystem列挙順を観測可能出力へ漏らさない。 | non-deterministic fixture failure |
| BR-U09-04 | duplicate identityまたは同じhost output pathを所有する複数artifactを許さない。 | collision一覧を返し、全projectionを拒否 |
| BR-U09-05 | plugin専用の第二stage/manifest parserを作らず、C1 normalized contractだけを消費する。 | schema driftとして差戻し |

公開seamは`unit-of-work.md`が列挙する4関数とし、各signatureは`component-methods.md`へ一致させる。`buildSelfInstallProjection`はC5内部helperであり、第五のpublic seamではない。公開関数の引数を`ValidPluginSource`等の別型へ置換しない。

## Projection and ownership rules

| ID | Rule | Failure behavior |
|---|---|---|
| BR-U09-06 | `plugins/<name>/`はauthoring source、`dist/plugins/<name>/`とhost treeはgenerated projectionである。 | generated bytesをsource扱いする実装を拒否 |
| BR-U09-07 | pluginは既存`HarnessManifest`のpath/token/rules/frontmatter変換を通す。harness別copyロジックをpluginへ複製しない。 | 6面driftまたは保守性違反として差戻し |
| BR-U09-08 | manifestを持つ全6 harnessへ同一source snapshotを投影する。 | 1面でも未生成ならbuild全体を失敗 |
| BR-U09-09 | package treeと`dist/plugins/`はgeneratorだけが書き、手編集を正解として吸収しない。 | DIFFERS/ORPHANとして検出 |
| BR-U09-10 | U09はhost composition record、doctor state、runtime graphを変更しない。 | U10 ownership侵害として差戻し |

## Compatibility and self-install rules

| ID | Rule | Failure behavior |
|---|---|---|
| BR-U09-11 | plugin 0件では既存`buildTree`と6 harness出力・CLI resultをbyte-identicalに保つ。 | golden baseline failure |
| BR-U09-12 | plugin固有path/indexはplugin 1件以上のときだけ生成する。 | empty projectionによるNFR-3違反 |
| BR-U09-13 | self-install対象は`claude | codex | cursor | opencode`のclosed unionである。 | 未知値、kiro、kiro-ideをcompile/runtimeで拒否 |
| BR-U09-14 | kiro、kiro-ideは6面packageに含めるがself-installへ昇格しない。 | project-local unexpected pathとして拒否 |
| BR-U09-15 | 6面package checkと4面self-install checkを独立実行し、結果を混同しない。 | verification evidence不足 |

## Drift and failure rules

| ID | Rule | Failure behavior |
|---|---|---|
| BR-U09-16 | temp buildにありcommitted distにないpathは`MISSING`、bytes差は`DIFFERS`である。 | check非0、write 0 |
| BR-U09-17 | committed generated treeにだけあるpathは`ORPHAN`である。 | check非0。write modeはownership内だけsweep |
| BR-U09-18 | discoveryしたauthoring sourceがbuild read-setにない場合は`UNREFERENCED`である。 | check非0、sourceは削除しない |
| BR-U09-19 | validationまたはcollision failure後に成功harnessだけをcommitしない。 | dist/self-install全体のpre-run bytes比較で拒否 |
| BR-U09-20 | drift listはharness、kind、path順にcanonical sortし、先頭失敗だけで残りを隠さない。 | snapshot/diagnostic driftとして拒否 |
| BR-U09-21 | 配布frameworkへruntime dependency、credential、network fetchを追加しない。 | supply-chain boundary違反 |

## Traceability and verification

- BR-U09-01〜05は`unit-of-work.md`のdiscovery seamと`component-methods.md`の`discoverPluginSources`へtraceする。
- BR-U09-06〜10は`components.md`のC5 ownershipと`services.md`のauthor→validate→project workflowへtraceする。
- BR-U09-11〜15は`requirements.md`のNFR-3/NFR-4、6 package/4 self-install境界へtraceする。
- BR-U09-16〜21はFR-6 item 19、NFR-1/NFR-8、`checkHarnessTree`のbyte/orphan/unreferenced contractへtraceする。
- `unit-of-work-story-map.md`に従い、U09はitem 19のtargeted testsを所有し、`test-pro` source/e2eはU11、全item ledger判定はU12へ残す。

|---|---|
| `unit-of-work.md` | BR-U09-01〜05、16〜20の公開seamとUnit境界 |

| `components.md` | BR-U09-06〜10のC5 ownerとgenerator-only write |
