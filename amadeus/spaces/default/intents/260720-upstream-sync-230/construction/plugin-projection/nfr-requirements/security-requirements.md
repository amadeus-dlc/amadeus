# Security Requirements — plugin-projection

> 根拠: `business-logic-model.md`、`business-rules.md`、`requirements.md`、`technology-stack.md`。plugin source、manifest、relative pathをuntrusted build inputとしてvalidation-before-writeを徹底する。

## Input・ownership controls

| ID | 脅威 | control | 合格条件 |
|---|---|---|---|
| SEC-U09-01 | unsafe source/path | `plugins/<name>/`内のC1検証済みidentity/artifact pathだけを使う。 | root外path fixtureでwrite 0。 |
| SEC-U09-02 | duplicate/collision | duplicate identityと同一host output ownership衝突を全build前に拒否する。 | collision一覧と全projection不変。 |
| SEC-U09-03 | generated-source confusion | authoring source、dist/plugins、host projectionを別identityで追跡し、generated bytesをsourceへ読み戻さない。 | dist手編集をDIFFERS/ORPHAN検出。 |
| SEC-U09-04 | unauthorized self-install | closed union `claude|codex|cursor|opencode`だけをpromoteし、kiro/kiro-ideを拒否する。 | project-local unexpected path 0。 |
| SEC-U09-05 | partial distribution | 全validation/temp build完了前にcommitted dist/self-installへ書かない。 | failure injectionでpre-run bytes一致。 |

plugin固有の第二schema/parser、harness別copy logic、compose/doctor/drop mutationをU09へ追加しない。

## Supply chain・compliance

- runtime dependency、credential、network fetch、database、service、UIを追加しない。
- `dist/`とproject-local self-installはgeneratorだけが書き、手編集を正解として吸収しない。
- upstream/source pluginをbuild inputとしてvalidateし、任意code execution面を作らない。
- new audit event、retention、permission、marketplaceを追加しない。

追加規制要件はない。既存license、human approval、audit boundaryを維持する。

## Failure behavior

malformed、duplicate、unsafe、collisionはloud failureかつwrite 0。check modeはMISSING/DIFFERS/ORPHAN/UNREFERENCEDを返してwrite 0。write modeのsweepはgenerator ownership内だけに限定する。

## トレーサビリティ

SEC-U09-01〜05は`business-rules.md`のBR-U09-01〜21、`business-logic-model.md`のOwnership/Failure table、`requirements.md`のNFR-2/4/8、`technology-stack.md`に対応する。
