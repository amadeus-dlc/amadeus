# Security Design — plugin-projection

> 上流入力(consumes全数): `performance-requirements.md`、`security-requirements.md`、`scalability-requirements.md`、`reliability-requirements.md`、`tech-stack-decisions.md`、`business-logic-model.md`。

## 信頼境界と入力検証

`plugins/<name>/`のdirectory名、manifest、artifact relative pathはuntrusted build inputとして扱う。repository root直下だけを発見対象とし、C1の既存validation resultでidentity、path、shapeを検証する。root外path、malformed manifest、duplicate identityは全pluginのbuild/write開始前にloud failureとする。第二schema、第二parser、generated treeからauthoring sourceへの逆流は追加しない。

output ownershipは全期待pathの構築時に検査し、core、別plugin、別artifactが同一host pathを所有するcollisionをcommit前に拒否する。source、`dist/plugins/`、host projectionは別identityとして追跡し、手編集されたgenerated bytesを正本へ昇格させない。

## 配布・mutation統制

package対象はclaude、codex、cursor、kiro、kiro-ide、opencodeの6面である。self-install対象はclaude、codex、cursor、opencodeのclosed 4面だけとし、kiro/kiro-ideのproject-local promotion要求は型境界で拒否する。writeはgenerator ownership内に限定し、check modeは常にwrite 0とする。

6 harness treeはtemp rootへ全生成し、全検査が成功した後にだけcommitする。validationまたはprojection failure時はdistとself-installの事前bytesを保持する。credential、network fetch、任意code execution、database、service、UI、marketplace、audit event、retention policyは追加しない。

## Failure control

| 状態 | 判定 | mutation |
|---|---|---|
| malformed・duplicate・unsafe | loud validation failure | 0 |
| output collision | loud collision failure | 0 |
| missing/different | `MISSING` / `DIFFERS` | checkは0 |
| generated-only | `ORPHAN` | checkは0、writeはownership内sweepのみ |
| discovered source未参照 | `UNREFERENCED` | source削除0 |
| kiro系self-install | closed-union failure | project-local tree不変 |

## トレーサビリティ

本設計は`security-requirements.md`のSEC-U09-01〜05を中心に、`performance-requirements.md`の全件検証、`scalability-requirements.md`のclosed inventory、`reliability-requirements.md`のfailure containment、`tech-stack-decisions.md`の既存packager/promoter再利用、`business-logic-model.md`のOwnership modelへ対応する。
