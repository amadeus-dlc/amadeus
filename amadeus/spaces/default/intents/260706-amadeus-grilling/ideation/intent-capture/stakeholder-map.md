# Stakeholder Map — Amadeus Grilling 統合

**Intent**: mattpocock の grilling スキルを Amadeus Grilling として統合する
**Date**: 2026-07-06 / **Stage**: intent-capture (1.1)

## Key Stakeholders

| ステークホルダー | 関心事 | 種別 |
|---|---|---|
| リポジトリオーナー(j5ik2o) | 対話品質の向上、ドッグフーディングの成功、フレームワークの一貫性維持 | 意思決定者 |
| Amadeus 導入チーム(外部ユーザー) | どのハーネスでも同じ grilling 体験、既存ワークフローへの非破壊的追加 | 受益者(間接) |
| mattpocock(元スキル作者) | MIT ライセンスに基づく適切な帰属表示 | 外部・帰属対象 |
| フレームワーク保守者(将来のコントリビュータ) | stage-protocol の複雑性増加を最小限に、4ハーネスのパリティ維持 | 影響者 |

## Decision-Makers vs. Influencers

- **意思決定者**: リポジトリオーナーのみ。各ステージの承認ゲートで判断する。単一チーム・単一スペース(spaces/default)構成のため、合議体は存在しない。
- **影響者**: 将来の保守者とハーネスのパリティ制約(package.ts の配布契約、`--check` ドリフトガード、t68 等のテスト群)。人間ではないが、設計の自由度を実質的に制約するステークホルダー相当の力を持つ。

## Communication Requirements

- **ステージゲート**: すべての承認はこのワークフローの承認ゲート(AskUserQuestion)経由。外部への報告義務はない。
- **帰属表示**: 取り込むスキルファイル内の MIT 帰属コメントと docs のクレジット記載が、元作者への唯一のコミュニケーション面。リリース時に CHANGELOG にも由来を明記する。
- **外部ユーザー向け**: docs/guide の対話モード説明(現3モードの記載箇所)に Grill me を追記し、配布物のオンボーディングドキュメントから発見可能にする。
