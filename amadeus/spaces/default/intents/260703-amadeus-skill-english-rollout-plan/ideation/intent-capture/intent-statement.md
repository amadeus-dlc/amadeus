# Intent Statement：Amadeus skill 英語化実施計画

## 目的

Amadeus skill の `SKILL.md` 英語化を、小さい土台 PR から段階的に進め、全面英語化の完了まで管理する。

親 Issue #399 を起点に、#395、#400、#401、#402 の順序、依存関係、完了境界、完了状態を追跡したうえで、#391〜#394 の差分対応と残り Amadeus skill の段階的英語化が完了したことを追跡できる状態にする。

## 解決する課題

Maintainer が、Amadeus skill 英語化の子 Issue の順序、依存関係、完了境界を追いにくい。

英語化を親タスク管理だけで閉じると、子 Issue の実施と完了確認が細切れになりすぎる。

## 対象者

| 対象者 | 役割 | 痛み |
|---|---|---|
| Maintainer | Amadeus 本体の方針、PR、merge 判断を行う。 | 子 Issue の順序、依存関係、完了境界、完了状態を追いにくい。 |
| Agent | skill、validator、example、docs の変更を提案または実行する。 | どの Issue をどの順序で扱うべきかを見失いやすい。 |
| Reviewer | PR の妥当性、検証、コメント対応を確認する。 | 英語化 PR の意味変更と翻訳差分の境界を判断しにくい。 |

## 成功条件

| ID | 成功条件 | 観測方法 |
|---|---|---|
| SC001 | #395、#400、#401、#402 の順序と依存関係を追跡できる。 | Issue #399、Intent 成果物、後続の計画成果物から確認する。 |
| SC002 | #395、#400、#401、#402 の完了状態を追跡できる。 | 対応 PR の merge または明示的な Issue close を完了証拠にする。 |
| SC003 | #401 配下の #391、#392、#393、#394 の扱いを追跡できる。 | #401 の完了証拠として、#391〜#394 の扱いが記録されていることを確認する。 |
| SC004 | Amadeus 系 `SKILL.md` の全面英語化が完了したことを追跡できる。 | `skills/amadeus*/SKILL.md` と `.agents/skills/amadeus*/SKILL.md` の英語化 PR、昇格フロー、検証結果から確認する。 |

## 完了境界

この Intent は、#395、#400、#401、#402 の完了だけでは閉じない。

Issue #399 の完了境界には、#391〜#394 の差分対応と、残り Amadeus 系 `SKILL.md` の全面英語化完了まで含める。

子 Issue の完了は、対応 PR の merge または明示的な Issue close で観測する。

#401 は #391〜#394 の扱いと実施順序を確定する計画単位である。

#391〜#394 個別の完了または対象外判断は、Issue #399 の残タスクとして追跡する。

## 契機

子 Issue の完了だけで閉じると、Amadeus skill 英語化の作業が細切れになり、親 Issue #399 が本来管理する全面英語化の完了確認が失われる。

そのため、親 Issue #399 を起点に、差分対応、残り skill の段階的英語化、最終検証まで一体で追跡する Intent として扱う。

## 範囲
