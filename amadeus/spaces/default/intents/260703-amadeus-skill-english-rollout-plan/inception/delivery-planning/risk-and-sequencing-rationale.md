# Risk and Sequencing Rationale：Amadeus skill 英語化実施計画

## 概要

この成果物は、Bolt の順序付けの根拠とリスクを整理する。

順序付けは依存先行とする。

## 順序付けの根拠

| 順序 | Bolt | 根拠 |
|---:|---|---|
| 1 | B001 #395 方針確定 Bolt | 後続 Bolt が英語化方針、対象範囲、検証方法を前提にするため。 |
| 2 | B002 #400 小さい土台 PR Bolt | 小さい土台 PR の結果が、#401 の AI-DLC v2 差分対応順序の前提になるため。 |
| 3 | B003 #401 AI-DLC v2 差分対応順序 Bolt | #391、#392、#393、#394 の扱いが、残り展開単位の前提になるため。 |
| 4 | B004 #402 残り展開単位 Bolt | #395、#400、#401 の完了証拠がそろった後に、残り skill の段階的英語化単位を決めるため。 |
| 5 | B005 #391〜#394 AI-DLC v2 差分対応 Bolt | #401 で確定した順序に沿って、個別 Issue の完了または対象外判断を確認するため。 |
| 6 | B006 Core entrypoints and verification 英語化 Bolt | 後続 skill が参照する入口、Space 初期化、検証の語彙を先にそろえるため。 |
| 7 | B007 Construction stage skills 英語化 Bolt | 代表 skill と同じ phase へ英語化方針を展開するため。 |
| 8 | B008 Inception stage skills 英語化 Bolt | Construction へ渡す語彙をまとめてそろえるため。 |
| 9 | B009 Ideation and supporting skills 英語化 Bolt | 入口と補助 skill の語彙を、Core と phase skill に合わせるため。 |
| 10 | B010 #399 最終検証 Bolt | 全面英語化完了後にだけ、親 Issue の完了判断へ進むため。 |

## リスク

| ID | リスク | 影響 | 対応 |
|---|---|---|---|
| RSK001 | #395 の方針が未確定のまま #400 に進む。 | 土台 PR の翻訳変更と意味変更の境界が曖昧になる。 | B001 を walking skeleton とし、先に方針と検証方法を確定する。 |
| RSK002 | #400 の結果を待たずに #401 を進める。 | AI-DLC v2 差分対応順序が土台 PR の知見を反映できない。 | B002 完了後に B003 へ進む。 |
| RSK003 | #401 配下 Issue を #401 の計画証拠だけで完了扱いにする。 | #399 が差分対応未完了のまま閉じられる。 | #391、#392、#393、#394 は B005 で個別完了または対象外判断を確認する。 |
| RSK004 | #402 が早すぎる。 | 残り展開単位が #401 の判断を反映できない。 | B003 完了後に B004 へ進む。 |
| RSK005 | #402 の完了を全面英語化完了と誤認する。 | `SKILL.md` が未英語化のまま #399 が閉じられる。 | #402 は展開単位の確定までとし、RU002〜RU006 の実行を B006〜B009 で追跡する。 |

## 優先方針

価値先行やリスク先行より、依存先行を採用する。

理由は、Issue #399 の成功条件が、直接子 Issue の順序と依存関係に加えて、差分対応と全面英語化の完了まで追跡できることにあるためである。
