# Requirements：Amadeus skill 英語化実施計画

## 概要

この成果物は、Issue #399 の成功条件を検証可能な要求へ落とす。

各要求は、#395、#400、#401、#402 の順序、依存関係、完了証拠、#391〜#394 の差分対応、残り `SKILL.md` の全面英語化、親 Issue の完了判断へ追跡できる。

受け入れ条件は各要求に内包する。

## 要求一覧

| ID | 要求 | 由来する成功条件 |
|---|---|---|
| R001 | #395、#400、#401、#402 の順序と依存関係を固定して追跡できる。 | SC001 |
| R002 | #395、#400、#401、#402 の完了証拠を要求の受け入れ条件として定義し、後続の delivery-planning と traceability で更新対象にできる。 | SC002 |
| R003 | #401 配下の #391、#392、#393、#394 の扱いを、#401 の計画証拠として追跡できる。 | SC003 |
| R004 | skill 英語化 PR で、翻訳変更、意味変更、昇格フロー、検証結果の境界を追跡できる。 | SC001、SC002 |
| R005 | #399 親タスクの完了判断を、子 Issue の完了証拠と全面英語化の完了証拠からできる。 | SC001、SC002、SC003、SC004 |
| R006 | Amadeus 系 `SKILL.md` の全面英語化完了を、段階的な PR 単位と最終検証で追跡できる。 | SC004 |

## R001：子 Issue の順序と依存関係

### 要求

#395、#400、#401、#402 は、この順序で扱う。

依存関係は、#400 が #395 に依存し、#401 が #400 に依存し、#402 が #401 に依存するものとして追跡する。

### 受け入れ条件

- #395、#400、#401、#402 の順序が要求成果物から確認できる。
- 各子 Issue の依存関係が要求成果物から確認できる。
- 順序または依存関係を変える場合は、requirements と traceability の更新対象として扱う。

## R002：子 Issue の完了証拠

### 要求

#395、#400、#401、#402 の完了証拠を、要求の受け入れ条件として定義する。

完了証拠は、対応 PR の merge または明示的な Issue close とする。

### 受け入れ条件

- #395、#400、#401、#402 の各要求に、対応 PR の merge または明示的な Issue close が完了証拠として記録されている。
- delivery-planning で、各子 Issue の進行単位へ完了証拠の更新先を割り当てられる。
- traceability で、各子 Issue の完了証拠の記録先を持てる。

## R003：#401 配下 Issue の扱い

### 要求

#391、#392、#393、#394 は、#401 で確定した順序と PR 境界を前提に追跡する。

#401 の完了は、#391〜#394 の扱いを確定した証拠であり、#391〜#394 個別の完了を代替しない。

### 受け入れ条件

- #401 の受け入れ条件に、#391、#392、#393、#394 の扱いを確認する条件が含まれている。
- #401 の計画成果物から、#391、#392、#393、#394 の扱いと実施順序を確認できる。
- #391、#392、#393、#394 の個別完了または対象外判断を、Issue #399 の残タスクとして追跡できる。

## R004：skill 英語化 PR の境界

### 要求

skill 英語化 PR では、翻訳変更、意味変更、source skill と昇格先 skill の同期、検証結果の境界を追跡する。

source skill から昇格先 skill への反映は、昇格フローを使う。

### 受け入れ条件

- PR 説明に、対象 Issue、対象 Intent、変更範囲、検証結果が含まれている。
- 翻訳変更と意味変更の境界が、PR 説明または関連成果物から確認できる。
- source skill と昇格先 skill の同期が必要な場合は、昇格フローを使ったことが確認できる。

## R005：親 Issue の完了判断

### 要求

#399 は、#395、#400、#401、#402 の完了証拠だけでは完了判断しない。

#399 は、#391〜#394 の差分対応と Amadeus 系 `SKILL.md` の全面英語化が完了した証拠をもとに完了判断する。

merge 操作そのものは人間が行う。

### 受け入れ条件

- #395、#400、#401、#402 の完了証拠がそろっていても、全面英語化が未完了なら #399 を完了扱いにしない。
- #391、#392、#393、#394 の完了または対象外判断を確認できる。
- `skills/amadeus*/SKILL.md` と `.agents/skills/amadeus*/SKILL.md` の英語化完了を確認できる。
- Agent は merge を実行せず、人間の merge を完了証拠として扱う。

## R006：全面英語化の追跡

### 要求

Amadeus 系 `SKILL.md` の全面英語化を、`docs/amadeus/skill-englishization-rollout-plan.md` の RU002〜RU006 に沿って追跡する。

source skill から昇格先 skill への反映は、昇格フローを使う。

### 受け入れ条件

- RU002〜RU006 の各 PR で、対象 skill、翻訳変更、意味変更、昇格フロー、検証結果を確認できる。
- `skills/amadeus*/SKILL.md` と `.agents/skills/amadeus*/SKILL.md` に残る日本語がないことを確認できる。
- 日本語維持対象である `aidlc/**/*.md`、templates、ユーザー向け gate 文言は変更対象から外している。
