# User Stories Assessment — installer-distribution

> ステージ: user-stories (2.4) / 作成: 2026-07-08

## 判定

**Execute(実行する)**

## 根拠

- **ユーザー対面機能である**: `@amadeus-dlc/setup` は外部 OSS 利用者が最初に触れる入口(intent-statement: 配布経路はユーザー体験の入口)。CLI の対話フロー・エラー体験・案内文がプロダクト価値そのもの
- **複数ペルソナが存在する**: 新規ユーザー(未導入・1コマンド導入)、既存ユーザー(カスタマイズ保持更新)、メンテナ(publish 運用)の3者で目標と痛点が異なる(stakeholder-map)
- **分岐の多い業務ロジック**: 導入済み検出・衝突・md5 判定・バージョン境界(FR-004/005/008)はペルソナ視点の受け入れ基準で検証可能性が上がる

## ストーリーが最も価値を生む領域

1. install ジャーニー(対話/非対話の分岐、導入済み検出)
2. upgrade ジャーニー(差分レポート確認、カスタマイズ保持、バックアップ)
3. メンテナの publish ジャーニー(手順書、タグ発行、pack 検証)

## 代替カバレッジの評価

requirements.md(FR 18件、Given/When/Then 受け入れ基準付き)は契約として十分精密だが、ペルソナ横断のジャーニー(「新規ユーザーが1分で開発を始められる」)は FR 単体では表現されないため、ストーリー層に価値がある。
