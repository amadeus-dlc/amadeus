> 提出先: 次回ローリング PM ラウンド(blind 選挙 → 採用ならノルム PR で team.md Way of Working へ)。
> 実測根拠: 2026-07-11 のレートリミット停止/再開の実運用(全6メンバー+leader の pause/park/復帰)、
> e1 ボトルネックレポート(可用性ギャップ≒6時間/日が最大滞留源、改善候補 (a)(b))、

## 候補1: 一時停止プロトコル(leader 配信の標準形式)

ユーザーからの停止指示(レートリミット接近等)を受けたら、leader は全メンバーへ以下の標準形式で配信する:

1. **即停止**: 優先度の低い作業(探索・クロスレビュー・PM 材料準備等、ユーザーが指定した範囲)。
   着手済みで途中のものは中断メモ(検証済みエビデンス・再開手順・投稿ドラフト等)を自分の record/scratchpad に保存して停止。
2. **最小 checkpoint まで許容**: 進行中の実装・ゲート工程は、成果を失わない最小の区切り
   (wip コミット+push+park)まで進めて停止。新しいステージ・unit・レビューには入らない。
3. **停止後**: 待機。自発再開禁止。再開はユーザー指示を受けて leader が配信する。
4. **監視維持**: agmsg monitor は受信のみで維持してよい。
5. **leader 自身**: 自 intent を park し、監査シャード込みで checkpoint コミット+push。
   全メンバーの ack(park コミット SHA 付き)を回収して停止完了とする。

## 候補2: 再開リカバリ・プレイブック(leader の標準手順)

再起動・復帰後(leader 再起動、全員再起動、部分再起動のいずれでも)、leader は以下を順に実行する:

1. **監視復旧**: agmsg monitor を再アーム(SessionStart hook の指示に従う)。
2. **未読消化**: inbox バックログを時系列で読む。ただしバックログは「当時の申告」であり現在の状態ではない。
3. **実測照合**: 現在地は必ず一次データで確定する — origin の各ブランチ(park コミット)、
   GitHub(open PR / Issue / CI)、intent record(amadeus-state.md / 停止時サマリ)。
   バックログ記載のタスクが解決済みでないか必ず突合する(ステール指示の再実行防止)。
4. **生存確認**: 全メンバーへ ping し、返信(実応答)で生存を確定する(send 成功は生存の証拠にならない)。
   無応答メンバーには割当を積まない(不達キュー退避)。再起動が必要なら identity ランナーでの起動を
   ユーザーに案内する(AI はライフサイクル操作禁止)。
5. **ステール訂正**: 各メンバーの再開時、再起動前の古いインボックス指示を実測ベースで訂正してから
   現在の正しい仕掛かりを再割当する。
6. **優先順で再開配信**: 停止時サマリの優先順(なければ: レビュー残 > 進行中 intent の着地 > 新規)で再開を配信。
   限定再開(1 intent のみ等)の指定があればそれに従う。
7. **human-presence の注意**: 再開後の delegate 発行はゲート開放後のユーザー実 HUMAN_TURN が必要になる
   場面がある。leader は必要時にユーザーへ一言依頼し、承認系の問いと束ねて HUMAN_TURN を効率利用する。
   HUMAN_TURN を含むシャードは delegate と同時にコミット+push する(参照先不在拒否の防止)。

## 候補3: ユーザー側の再開契約(1行で足りる)

ユーザーは leader へ「レートリミットから復帰。(全員/一部)再起動済み。リカバリして再開して」の
1行を送れば、候補2の全手順が実行される。オプション: 「◯◯だけ再開」(限定再開)/「余裕少なめ」
(軽量モード: 1行選挙・最小報告)/「全開でよい」。

## 補足(選挙時の判断材料)

- 候補1/2 は 2026-07-11 に実際に運用して成功した手順の成文化(停止: 全員 ack・成果ゼロ損失、
  復帰: 誤再実行ゼロ、限定再開: complexity-gate を最小コストで close)。
- 既存ノルムとの関係: c5(無応答引き取り)・agmsg-recipient-typo(不達診断)・checkpoint 規範・
  no-session-lifecycle-ops と整合し、それらを「停止/再開」の文脈に束ねる上位手順。矛盾なし。
- 軽量選挙形式(1行返信)の恒久化は本候補に含めない(停止中の特例として都度 leader 判断。
  含めるべきかは選挙で異議として提起可)。
