## Q1. CLI の表示言語はどうしますか?

- A. 英語のみ(OSS の標準。ドキュメントも英語)
- B. 英語デフォルト + 日本語オプション(`--lang ja`、cc-sdd と同様)
- C. 日本語のみ
- X. Other (please specify)

[Answer]: B — 英語デフォルト + 日本語オプション(--lang ja)(2026-07-07, Mode: guided)

## Q2. 対話式ウィザードのステップ構成はどこまで聞きますか?

- A. 最小 — ハーネス選択のみ(1問)。他はデフォルト+フラグ
- B. 標準 — ハーネス選択 + インストール先確認 + 実行前サマリー確認(2〜3ステップ)
- C. 丁寧 — B + バージョン選択 + 上書きポリシー確認(4〜5ステップ)
- X. Other (please specify)

[Answer]: B — 標準(ハーネス選択 + 導入先確認 + 実行前サマリー、2〜3ステップ)(2026-07-07, Mode: guided)

## Q3. 出力スタイルの方向性は?

- A. ミニマル — プレーンテキスト、記号(✓ / !)程度
- B. リッチ — 色付き・進捗スピナー・整形されたテーブル(差分レポート)
- C. ミニマル基本 + 差分レポートのみテーブル整形
- X. Other (please specify)

[Answer]: C — ミニマル基本 + 差分レポートのみテーブル整形(2026-07-07, Mode: guided)

## 質問圧縮とアクセシビリティ方針の記録(設計判断)

ステージ例示の7トピックのうち「アクセシビリティ要件」「デバイス/フォームファクタ」「情報階層」「既存ブランド/デザインシステム」の4項目は質問化しなかった。理由: CLI プロダクトのため GUI 前提のトピック(フォームファクタ・ブランドシステム)は適用外であり、情報階層は Q2/Q3 の決定に包含される。アクセシビリティはユーザーに問うのではなく次の CLI 向け方針を設計側で規定した(wireframes.md 冒頭に明記): (1) 色のみに依存しない記号+文言併記、(2) 行単位テキストで読み上げ順=表示順、(3) キーボードのみで完結(矢印選択に番号入力の代替)。WCAG の GUI 規約は適用外とする。
