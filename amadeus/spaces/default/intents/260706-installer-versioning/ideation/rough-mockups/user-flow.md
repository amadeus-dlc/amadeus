# User Flow — 260706-installer-versioning（Issue #543）

上流入力: [wireframes.md](wireframes.md)

## フロー 1: 通常更新（改変なし）

1. 利用者が従来と同じ 1 コマンドを再実行する。
2. インストーラが manifest を読み、3-way 判定で全ファイル「未改変」→ 従来どおり収束コピー。導入先で削除されていたファイルは再作成し、件数を告知する（協議 Q4）。
3. manifest を新しい commit / 時刻 / ハッシュ表で書き直す。
4. 改変・削除がなければ出力は従来と同一（退避行・restored 行なし）。

## フロー 2: カスタマイズありの更新

1. 同じ 1 コマンドを再実行する。
2. 3-way 判定で改変検出 → 集中退避 dir へ相対 path 保存で退避 → 新版で上書き。
3. summary に退避一覧を列挙（無言にしない）。
4. 利用者は退避物と新版を突き合わせ、必要な変更を自分で再適用する（インストーラは 3-way merge をしない = Right-Sizing）。

## フロー 3: 版の確認

1. 利用者が版確認の入口（flag / subcommand）を実行する。
2. manifest の commit / 導入時刻 / 追跡ファイル数を 1 行で得る。manifest 不在なら不在の旨を 1 行で得る。

## フロー 4: bootstrap（manifest 不在の既導入環境の更新）

1. 同じ 1 コマンドを再実行する。
2. manifest 不在 → 記録なしファイルは「新配布物と不一致なら退避して上書き」（協議 Q6 = 保守的）。
3. 以後は manifest が存在するため通常の 3-way に移行する。
