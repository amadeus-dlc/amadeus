# Component Methods：Amadeus skill 英語化実施計画

## 概要

この成果物は、論理コンポーネントのメソッド境界を整理する。

ここでのメソッドは実装関数ではなく、後続 stage が参照する責務単位である。

## メソッド境界

| ID | コンポーネント | メソッド境界 | 入力 | 出力 |
|---|---|---|---|---|
| M001 | C001 | 子 Issue 順序を確定する。 | #395、#400、#401、#402 | 順序と依存関係 |
| M002 | C001 | 順序変更を更新対象として示す。 | 順序変更の判断 | requirements と traceability の更新対象 |
| M003 | C002 | 子 Issue の完了証拠を判定する。 | PR merge または明示的な Issue close | 子 Issue の完了状態 |
| M004 | C003 | #401 配下 Issue の扱いを確認する。 | #391、#392、#393、#394 の扱い | #401 の完了証拠 |
| M005 | C004 | skill 英語化 PR の境界を確認する。 | PR 説明、変更範囲、検証結果 | Reviewer の判断材料 |
| M006 | C005 | #399 の完了判断可否を示す。 | #395、#400、#401、#402 の完了証拠 | 親タスクの完了判断可否 |

## 例外

完了証拠が確認できない子 Issue は、完了扱いにしない。

目的と異なるが有効な PR 指摘は、後続 Issue 候補として扱う。

merge 操作は Maintainer が行う。
