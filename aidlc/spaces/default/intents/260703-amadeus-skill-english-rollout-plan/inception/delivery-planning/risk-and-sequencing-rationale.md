# Risk and Sequencing Rationale：Amadeus skill 英語化実施計画

## 概要

この成果物は、Bolt の順序付けの根拠とリスクを整理する。

順序付けは依存先行とする。

## 順序付けの根拠

| 順序 | Bolt | 根拠 |
|---:|---|---|
| 1 | B001 #395 方針確定 Bolt | 後続 Bolt が英語化方針、対象範囲、検証方法を前提にするため。 |
| 2 | B002 #400 小さい土台 PR Bolt | 小さい土台 PR の結果が、#401 の AI-DLC v2 差分対応順序の前提になるため。 |
| 3 | B003 #401 AI-DLC v2 差分対応順序 Bolt | #391、#392、#393、#394 の扱いが、残り展開単位の前提になるため。 |
| 4 | B004 #402 残り展開単位 Bolt | #395、#400、#401 の完了証拠がそろった後に、残り skill の段階的英語化単位を決めるため。 |

## リスク

| ID | リスク | 影響 | 対応 |
|---|---|---|---|
| RSK001 | #395 の方針が未確定のまま #400 に進む。 | 土台 PR の翻訳変更と意味変更の境界が曖昧になる。 | B001 を walking skeleton とし、先に方針と検証方法を確定する。 |
| RSK002 | #400 の結果を待たずに #401 を進める。 | AI-DLC v2 差分対応順序が土台 PR の知見を反映できない。 | B002 完了後に B003 へ進む。 |
| RSK003 | #401 配下 Issue を親 Intent の直接完了条件へ広げる。 | #399 の完了境界が広がりすぎる。 | #391、#392、#393、#394 は B003 の完了証拠として扱う。 |
| RSK004 | #402 が早すぎる。 | 残り展開単位が #401 の判断を反映できない。 | B003 完了後に B004 へ進む。 |

## 優先方針

価値先行やリスク先行より、依存先行を採用する。

理由は、Issue #399 の成功条件が #395、#400、#401、#402 の順序と依存関係を追跡できることにあるためである。
