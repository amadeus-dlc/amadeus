# Interaction Spec：Amadeus skill 英語化実施計画

## 概要

この成果物は、Issue 完了追跡の相互作用を要求とストーリーに対応づける。

UI 操作は定義しない。

## 相互作用一覧

| ID | 相互作用 | 入力 | 出力 | 参照 |
|---|---|---|---|---|
| I001 | 子 Issue の順序確認 | #395、#400、#401、#402 | 順序と依存関係 | R001、S001 |
| I002 | 子 Issue の完了証拠確認 | 対応 PR merge または明示的な Issue close | 子 Issue の完了状態 | R002、S002 |
| I003 | #401 配下 Issue の扱い確認 | #391、#392、#393、#394 の扱い | #401 の完了証拠 | R003、S003 |
| I004 | skill 英語化 PR の境界確認 | PR 説明、変更範囲、検証結果 | Reviewer の判断材料 | R004、S004 |
| I005 | #399 完了判断 | #395、#400、#401、#402 の完了証拠 | 親タスクの完了判断 | R005、S005 |

## 正常フロー

1. Maintainer は #399 の対象子 Issue を確認する。
2. Agent は #395、#400、#401、#402 の順序と依存関係を成果物へ反映する。
3. Agent は各子 Issue の対応 PR merge または明示的な Issue close を完了証拠として確認する。
4. Agent は #401 の完了証拠として #391、#392、#393、#394 の扱いを確認する。
5. Reviewer は skill 英語化 PR の翻訳変更、意味変更、昇格フロー、検証結果の境界を確認する。
6. Maintainer は #395、#400、#401、#402 の完了証拠がそろった場合に #399 の完了判断へ進む。

## 例外フロー

子 Issue の順序または依存関係を変更する場合は、requirements と traceability の更新対象として扱う。

対応 PR merge または明示的な Issue close が確認できない子 Issue は、完了扱いにしない。

目的と異なるが有効な PR 指摘は、後続 Issue 候補として扱う。

merge 操作は Maintainer が行い、Agent は実行しない。
