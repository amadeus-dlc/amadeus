# UC002 マージ後に共有成果物を整合させる

## ユースケース

Agent が並行 branch のマージ後に、policy の統合手順に従って共有成果物を整合させる。

## アクター

- ACT002 Agent

## 外部システム

- なし

## 事前条件

- 並行していた branch のいずれかが基準 branch へマージされた。

## 基本フロー

1. Agent は最新の `origin/main` を取得し、作業 branch を追従させる。
2. 共有インデックス（`intents.md`、`discoveries.md`）に影響する変更がある場合は、再生成を実行して整合させる。
3. 標準検証で整合を確認してから作業を再開する。

## 代替フロー

| 条件 | 扱い |
|---|---|
| steering 共有資産の索引行が両 branch で追加されていた。 | 行追加同士の衝突として通常の conflict 解消で統合し、再生成対象であれば再生成を正とする。 |
