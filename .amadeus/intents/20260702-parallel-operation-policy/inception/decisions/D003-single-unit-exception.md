# D003: Intent と Unit の 1:1 例外

## 背景

ピラミッド構造では、下位成果物が上位成果物と常に 1:1 になる場合、分割不足を疑う。
この Intent は Unit が U001 の 1 件だけであり、Intent と Unit が 1:1 になっている。

## 判断

Intent と Unit の 1:1 を粒度例外として認める。

## 理由

並行可否、統合手順、承認運用、直列化の判断基準は相互に依存し（並行可否は統合手順の存在を前提にし、承認運用は並行の結果として生じる）、同じ policy 契約を共有する。
個別の判断基準を別 Unit に切っても、単独では「policy を根拠に並行作業を進められる」という受け入れ条件を満たせず、価値単位として成立しない。
要求は 5 件、ユースケースは 4 件、Bolt は 2 件に分かれており、下位の分解は 1:1 になっていない。

## 影響

後で分割を見直す条件は、判断基準群が独立に変更される必要が出た場合、または Bolt の依存 wave 並行実行（Issue #352）の契約化で並行実行の規則が別の価値単位として確定した場合である。
