# R002 共有成果物の統合手順

## 要求

並行 branch の統合時に共有成果物（共有インデックス、steering 共有資産）をどの順序で整合させるかが steering policy から読める。

## 背景

#334 の cycle で、共有インデックスの生成物化により「マージ後に `origin/main` を追従してから再生成する」統合手順が確立し、手動コンフリクト解消が不要になった実例が観察された。
Git Branching Policy は追従（merge 後処理）までを扱い、共有成果物の再生成の順序は扱っていない。

## 受け入れ条件

- マージ後の追従と共有インデックス再生成の順序が policy から読める。
- steering 共有資産（`policies.md`、README の索引など）への行追加が並行 branch と接触した場合の扱いが policy から読める。
- 手順の根拠が、観察した実例への参照リンクで追跡できる。
