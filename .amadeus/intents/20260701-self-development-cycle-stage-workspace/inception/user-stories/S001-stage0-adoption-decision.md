# S001: stage0 採用判断

## ストーリー

- Maintainer として、stage2 を次回 stage0 として採用できるかを、stage 判定と workspace 対応記録から判断したい。
- それにより、後続の自己開発 Intent がどの skill、validator、開発用スクリプトを基準として使うべきかを誤らずに判断できる。

- stage2 を stage0 として採用する条件が追跡できる。
- build workspace と target workspace の対応が追跡できる。
- validator と標準検証の結果が採用判断の証拠候補として追跡できる。
