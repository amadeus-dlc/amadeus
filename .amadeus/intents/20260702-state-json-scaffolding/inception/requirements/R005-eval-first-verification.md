# R005 eval 先行の検証

## 要求

雛形生成スクリプトの検証は、先に失敗する eval または決定論的検証を追加し、実装前に失敗（RED）を確認してから実装されている。

## 受け入れ条件

- 6 遷移それぞれの生成結果を固定入力で検証する eval が存在する。
- 実装前に eval が失敗した記録がある。
- eval は昇格先に混入しない場所（source 側の `evals/` または `dev-scripts/evals/`）に置かれ、repo の標準検証（`test:it:*`）から実行される。
- 生成直後の state に対する validator の pass が eval で確認される。
