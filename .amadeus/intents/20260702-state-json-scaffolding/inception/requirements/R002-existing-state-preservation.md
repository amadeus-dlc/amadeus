# R002 既存 state の保持

## 要求

雛形の更新は、既存 `state.json` の値と前 phase の状態ブロックを保持する。

## 背景

`state.json` は phase をまたいで累積する記録であり、ideation、inception の状態ブロックや taskGeneration の evidence は後続 phase でも参照される。
雛形が既存 state を上書きすると、承認証拠や完了記録を失う。

## 受け入れ条件

- 既存の値（完了済み phase のブロック、evidence、承認記録）を変更せずに、対象遷移のブロックだけを追加または更新する。
- 同じ遷移で複数回実行しても結果が変わらない（冪等である）。
- 上書きが必要な更新（status や gate の前進）は、対象遷移で定義された項目に限定される。
