# R001 統合条件と既定

## 要求

phase PR の統合を許可する条件と、許可しない場合の既定（phase ごとの PR）が Git Branching Policy から読める。

## 背景

小さい Intent でも phase ごとの PR と人間 merge を繰り返しており、直近の cycle では 2 Intent の完走に phase PR 13 本と人間 merge 13 回を要した。
統合を許可する条件が未定義のため、既定の運用しか選べない。

## 受け入れ条件

- 統合を許可する条件が、次の 3 条件すべてを満たす場合に限ることが policy から読める（G001 GD002）。
  - 実行スコープが `refactor` または docs 系である。
  - 変更対象が文書だけで、実装コードとテストコードを含まない。
  - Ideation の未確定事項が、事前の grilling または Issue の確定判断で解消済みである。
- 条件を満たさない場合の既定が phase ごとの PR であることが明記されている。
- 既存の phase ごとの PR 運用が引き続き有効である。

## 依存

なし。
