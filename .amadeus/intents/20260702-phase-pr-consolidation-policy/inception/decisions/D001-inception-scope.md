# D001: Inception の所有境界

## 背景

Issue #310 は policy 文書の変更だけで完結するが、統合条件の強度、単位、記載先は人間判断が必要だった。

## 判断

Inception の所有境界を、統合条件（3 条件必須）、統合単位（仕様側 2 グループ）、統合 PR の記録項目、既存文書との整合の要求、ユースケース、Unit、Bolt の定義に固定する。

policy の最終文言と節の配置は Construction で確定する。

## 理由

grilling の確定判断（GD001〜GD003）と Issue の対象、対象外、受け入れ条件から、契約の内容は確定できるが、文言は既存の Branch Lifecycle の節構造と整合させながら決める必要があるため。

## 影響

Construction の Functional Design では、統合条件の記述形式（判定基準の表形式か本文か）を最初に確定する。
