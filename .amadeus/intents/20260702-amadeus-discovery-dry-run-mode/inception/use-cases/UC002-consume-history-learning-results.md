# UC002: 過去分析と学習分類の結果を候補表示に使う

## システム境界

- Agent は必要に応じて、`amadeus-history-review` の過去分析結果または `amadeus-learning-review` の学習分類結果を `dry-run` の入力として扱う。
- `dry-run` は結果を候補表示に使うが、過去分析と学習分類そのものを所有しない。

## 事前条件

- `amadeus-history-review` または `amadeus-learning-review` の結果がある、またはそれらを入力候補として参照できる。
- `dry-run` の読み取り専用境界が定義されている。

## 基本フロー

1. Agent は過去分析結果または学習分類結果の有無を確認する。
2. Agent は分析結果の再利用判断、未確認事項、繰り返し問題、後続候補を候補表示の入力にする。
3. Agent は学習分類結果の後続 Issue 候補、後続 Intent 候補、戻り先を候補表示の入力にする。
4. `dry-run` は入力された結果を、Intent 候補、根拠、未確認事項、推奨次アクションへ反映する。
5. `dry-run` は分析と分類を所有しないことを説明できる状態で終了する。

## 代替フロー

- 分析結果や分類結果がない場合、`dry-run` は既存成果物と外部参照だけで候補表示を行う。
- 分析結果や分類結果に未確認事項がある場合、`dry-run` は未確認事項として表示し、成果物を推測で作らない。

## 事後条件

- 過去分析と学習分類の責務は `dry-run` に移っていない。
- `dry-run` の候補表示は、入力結果の根拠を追跡できる。

## BCE候補

| 種別 | 候補 | 責務 |
|---|---|---|
| 境界 | review result input | 過去分析結果と学習分類結果を候補表示の入力として受け取る。 |
| 制御 | consumer mapping | 分析結果と分類結果を候補表示へ写像する。 |
| エンティティ | referenced review result | 参照した分析結果、分類結果、根拠を保持する。 |

## 責務候補

| 候補 | 判断 | 保持 | 依頼 |
|---|---|---|---|
| `amadeus-discovery dry-run` | 入力された分析結果を候補表示へ使うかを決める。 | 参照した結果、候補、根拠。 | 過去分析は `amadeus-history-review`、学習分類は `amadeus-learning-review` に任せる。 |
