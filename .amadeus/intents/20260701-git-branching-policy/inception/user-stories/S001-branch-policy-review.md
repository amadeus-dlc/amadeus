# S001: branch policy review

## ストーリー

- Maintainer として、複数 Intent と複数 worktree の branch 判断を成果物から確認したい。
- それにより、PR 作成、merge 後処理、次作業の branch 切り替えが、GitHub 上の状態と Amadeus DLC 成果物の両方から追跡できる。

## 受け入れ条件

- branch 作成から PR 作成、merge 後処理までの判断が policy として読める。
- AGENTS.md の操作指示と steering policy の責務分担が読める。
- validator または evaluator で検出する候補と、人間判断に残す候補が読める。
