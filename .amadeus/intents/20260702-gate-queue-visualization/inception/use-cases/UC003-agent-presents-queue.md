# UC003 Agent が承認待ちを人間へ提示する

## ユースケース

並行作業中の Agent が、担当 Intent のゲートが承認待ちへ到達した際に一覧を実行し、workspace 全体の承認待ちキューを人間へ提示して承認を依頼する。

## アクター

- ACT002 Agent

## 外部システム

- なし

## 事前条件

- Agent が昇格済み成果物（配布先ユーザー環境相当）だけで一覧を実行できる。
- 少なくとも 1 件の Intent が承認待ち（Task Generation の `ready_for_approval` など）に到達している。

## 基本フロー

1. Agent は担当 Intent のゲートを承認待ちにし、停止する。
2. Agent は workspace を指定して一覧を実行する。
3. 一覧は担当 Intent を含む承認待ちの Intent、phase、ゲート、待ち理由を Markdown 表として表示する。
4. Agent は表を人間へ提示し、承認を依頼する。
5. 人間は表から承認対象を選び、承認後に Agent が作業を再開する。

## 代替フロー

| 条件 | 扱い |
|---|---|
| 他の Intent の承認待ちが同時に表示される。 | 人間はまとめて承認順序を判断でき、承認の見落としを避けられる。 |
