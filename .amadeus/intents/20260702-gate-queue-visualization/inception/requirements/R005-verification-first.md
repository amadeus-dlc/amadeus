# R005 検証の先行追加

## 要求

承認待ち判定と一覧出力（0 件時を含む）の検証を、実装より先に追加して RED を確認する。

## 背景

Ideation の検証戦略は standard であり、承認待ち判定の決定論的検証（RED → GREEN）を求めている。
スクリプト実装は dev-scripts ルールに従い、先に失敗する検証を書き、失敗を確認してから最小実装を入れる。

## 受け入れ条件

- 承認待ちを含む検証データ（`examples/04-construction-design-ready` 相当）と承認待ち 0 件の検証データに対する決定論的検証が、実装より先に追加されている。
- 追加した検証が実装前に失敗することが確認されている。
- 実装後に、追加した検証が成功する。

## 依存

R001、R002、R003。

- SC-IN-002
