# D003: Intent と Unit の 1:1 例外

## 背景

ピラミッド構造では、下位成果物が上位成果物と常に 1:1 になる場合、分割不足を疑う。
この Intent は Unit が U001 の 1 件だけであり、Intent と Unit が 1:1 になっている。

## 判断

Intent と Unit の 1:1 を粒度例外として認める。

## 理由

スキャン、判定、出力、配布、検証は同じ一覧契約を共有し、独立に価値を提供できない。
判定だけ、出力だけを別 Unit に切っても、単独では受け入れ条件を満たせず、価値単位として成立しない。
要求は 5 件、ユースケースは 3 件、Bolt は 2 件に分かれており、下位の分解は 1:1 になっていない。

## 影響

後で分割を見直す条件は、承認待ち判定の導出規約と一覧の出力契約が独立に変更される必要が出た場合、または一覧以外の利用形態（機械可読出力、他ツールへの組み込み）が要求になった場合である。
