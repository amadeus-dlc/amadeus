# R004: 互換性境界

## 要求

互換性維持対象が明示されていない場合に、旧入口、旧名、alias、互換層を追加しない判断を説明できる。

## 背景

このリポジトリの互換性ルールは、`docs/backward-compatibility.md` に記載された対象だけを互換性維持対象として扱う。
現在の target workspace では `docs/backward-compatibility.md` が存在しない。

そのため、README や skill 分類を調整するときに、暗黙の互換層を追加しない判断が必要である。

## 受け入れ状態

- 互換性維持対象の有無を確認できる。
- 互換性維持対象がない場合に、旧入口、旧名、alias、互換層を追加しないことを説明できる。
- 例外が必要な場合は、実装前に `docs/backward-compatibility.md` へ記録する必要があることを説明できる。
