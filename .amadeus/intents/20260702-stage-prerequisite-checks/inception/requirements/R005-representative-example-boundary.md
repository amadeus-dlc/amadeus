# R005 代表例の境界

## 要求

repo 内代表例と配布対象 skill の一般説明を分け、ユーザー環境で参照できない Issue 番号前提を混入させない。

## 背景

Issue #277 と Issue #272 の関係は、このリポジトリ内では前提不成立の代表例として有効である。

ただし、配布対象 skill の利用者は、このリポジトリの Issue 番号や経緯を参照できるとは限らない。

そのため、配布対象 skill では一般化した前提確認として説明する必要がある。

## 受け入れ条件

- repo 内成果物では、Issue #277 と Issue #272 の関係を代表例として説明できる。
- 配布対象 skill では、repo 内 Issue 番号を前提にしない一般説明へ置き換えられている。
- 代表例は、source skill の不足、昇格先成果物への未反映、host environment での利用不可のいずれかとして説明できる。
- eval、text contract、またはレビュー観点で、repo 内 Issue 番号前提の混入を確認できる。
