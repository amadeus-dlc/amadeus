# D003: Intent と Unit の 1:1 例外

## 背景

ピラミッド構造では、下位成果物が上位成果物と常に 1:1 になる場合、分割不足を疑う。
この Intent は Unit が U001 の 1 件だけであり、Intent と Unit が 1:1 になっている。

## 判断

Intent と Unit の 1:1 を粒度例外として認める。

## 理由

wave の導出、実行、統合、まとめ承認は同じ実行契約を共有し、相互に依存する（実行は導出を、まとめ承認は wave 単位の準備を前提にする）。
個別に Unit へ切っても、単独では「wave 単位で並行実行し、wave ごとに統合と検証を行う契約が読める」という受け入れ条件を満たせず、価値単位として成立しない。
要求は 4 件、ユースケースは 4 件、Bolt は 2 件に分かれており、下位の分解は 1:1 になっていない。

## 影響

後で分割を見直す条件は、wave 導出の機械化（解析スクリプト）が別の価値単位として必要になった場合、または wave 実行の運用実績から契約の分割が必要になった場合である。
