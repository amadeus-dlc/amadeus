# UC002 並行 branch を統合して再生成する

## ユースケース

並行する 2 つの branch がそれぞれ Intent または Discovery のモジュールを追加し、統合後の再生成で両方の行を含む共有インデックスを得る。

## アクター

- ACT002 Agent

## 外部システム

- なし

## 事前条件

- 各 branch は共有インデックスを手書き編集していない。

## 基本フロー

1. branch A の Agent は、Intent X のモジュールを追加する。
2. branch B の Agent は、Intent Y のモジュールを追加する。
3. 両 branch を基準 branch へ統合する。モジュールはファイル単位で分離しているため、コンフリクトは発生しない。
4. Agent は再生成スクリプトを実行し、Intent X と Intent Y の両方の行を含む `intents.md` を得る。

## 代替フロー

| 条件 | 扱い |
|---|---|
| 両 branch が再生成済みのインデックスを含んで統合され、インデックスがコンフリクトする。 | どちらの内容も採用せず、統合後に再生成して解消する。 |
