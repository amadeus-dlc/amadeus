# UC003 不整合を検出して解消する

## ユースケース

Agent が validator を実行し、共有インデックスと配下モジュールの不整合を fail として検出し、再生成で解消する。

## アクター

- ACT002 Agent

## 外部システム

- なし

## 事前条件

- validator の不整合検査が workspace 検証に含まれている。

## 基本フロー

1. Agent は workspace を指定して validator を実行する。
2. validator は、インデックスの行の過不足、行の内容と配下モジュールの不一致、生成マーカーの欠落を検査する。
3. 不整合がある場合、validator は対象と理由を fail として報告する。
4. Agent は再生成スクリプトを実行して不整合を解消し、validator の pass を確認する。

## 代替フロー

| 条件 | 扱い |
|---|---|
| インデックスが手書きで編集されている。 | 再生成結果との不一致として fail になり、再生成で上書き解消する。 |
| 配下モジュール側の見出しが不足している。 | validator または再生成スクリプトが不足を示し、Agent はモジュール側を補修する。 |
