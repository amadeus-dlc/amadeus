# UC001 モジュール変更時にインデックスを再生成する

## ユースケース

Agent が Intent または Discovery のモジュールを作成または更新した後、同梱スクリプトを実行して `intents.md` と `discoveries.md` を再生成する。

## アクター

- ACT002 Agent

## 外部システム

- なし

## 事前条件

- Intent のモジュールファイルが、概要と依存（理由付き）の見出しを持つ。
- 昇格済みの同梱スクリプトが配布先ユーザー環境から実行できる。

## 基本フロー

1. Agent は、Intent Record の作成または Discovery の作成で、モジュールファイルと `state.json` を書く。共有インデックスは手書きしない。
2. Agent は、workspace を指定して再生成スクリプトを実行する。
3. スクリプトは、`intents/` と `discoveries/` 配下のモジュールと `state.json` を読み、先頭マーカー付きの `intents.md` と `discoveries.md` を出力する。
4. Agent は、再生成されたインデックスをモジュールの変更と同じ変更に含める。

## 代替フロー

| 条件 | 扱い |
|---|---|
| モジュールの内容が変わらないまま再実行される。 | 出力は変わらない（冪等）。 |
| モジュールに概要または依存の見出しがない。 | スクリプトは不足しているモジュールと見出しを示して失敗する。 |
