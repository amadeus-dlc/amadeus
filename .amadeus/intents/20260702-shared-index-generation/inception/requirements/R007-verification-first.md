# R007 検証の先行追加

## 要求

再生成の決定論性と不整合検査の検証を、実装より先に追加して RED を確認してから実装する。

## 背景

Script Rules は、実装前に失敗する検証を追加し、失敗を確認してから最小実装を入れることを必須にしている。
再生成の決定論性（同じ入力から同じ出力）と並行統合の成立は、決定論的な検証として書ける。

## 受け入れ条件

- 再生成の決定論性を確認する検証が、実装より先に追加され、RED が確認されている。
- 並行統合のシナリオ（2 つのモジュール追加を統合して再生成すると両方の行を含む）が検証に含まれる。
- validator の不整合検査に対する検証（不整合を注入すると fail になる）が含まれる。

## 依存

R002、R004。
