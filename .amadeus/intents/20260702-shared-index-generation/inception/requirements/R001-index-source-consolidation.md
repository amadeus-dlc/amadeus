# R001 定義元の一元化

## 要求

各 Intent のモジュールファイルが、共有インデックスの行を構成する概要と依存（理由付き）を自分自身で持ち、`intents.md` と `discoveries.md` にしか存在しない情報がなくなる。

## 背景

現状、Intent の概要、依存、依存理由は `intents.md` にしか書かれていない。
grilling の確定判断 GD001 により、これらの定義元を各 Intent のモジュールファイルへ移し、共有インデックスを完全な生成物にする。
Discovery のモジュールファイルは、テーマ、状態、判定、推奨次アクションを既存の見出しと `state.json` から導けるため、新しい見出しの追加は原則不要である。

## 受け入れ条件

- Intent のモジュールファイルの必須見出しに、概要と依存（理由付き）が含まれる契約が定義されている。
- `intents.md` の一覧行と依存関係行を構成するすべての情報が、配下モジュールまたは `state.json` から導出できる。
- `discoveries.md` の一覧行を構成するすべての情報が、Discovery のモジュールファイルまたは `state.json` から導出できる。

## 依存

なし。
