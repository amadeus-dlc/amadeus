# D003: 単一 Unit の粒度例外

## 背景

ピラミッド構造の基本形では、下位成果物が上位成果物と常に 1:1 になる場合は分割不足を疑う。
この Intent では Intent と Unit が 1:1 になっている。

## 判断

Intent と Unit の 1:1 を粒度例外として認める。

## 理由

定義元の移設、再生成、検査、手順更新、migration は、同じ生成契約（見出し契約と生成規則）を共有しており、別 Unit に分けると契約の合意を複数回やり直すリスクがある。
実施粒度は Bolt 4 件（B001 から B004）で分割しており、grill 不足ではなく単一の価値単位として自然な粒度である。
Requirement 7 件、Use Case 3 件、Bolt 4 件の分解は 1:1 ではない。

## 見直し条件

`glossary.md` など他の共有成果物の生成物化を扱う場合は、この Unit を拡張せず、別 Intent として扱う。
