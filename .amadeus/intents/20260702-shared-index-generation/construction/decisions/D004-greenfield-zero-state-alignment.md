# D004: greenfield のゼロ状態整合

## 背景

B003 の実装レビューで、steering テンプレートの index 雛形と `IndexGenerate.ts` のゼロ件出力が一致せず、greenfield の workspace が作成直後に「Index 生成整合」検査で fail する問題が見つかった。
テンプレートはゼロ状態の案内文（「Intent は未登録である。」等）を持ち、discoveries の H1 も生成器と異なっていた。

## 判断

生成器がゼロ件時にテンプレートと同じ案内文を出力する方向で揃える。

- `IndexGenerate.ts` は、一覧が 0 件のとき「Intent は未登録である。」「Intent 間の依存関係は未登録である。」「Discovery は未登録である。」を出力する。
- steering テンプレートの discoveries H1 を「# Discovery 一覧」から「# Discovery」へ変更し、生成器と実ファイルに合わせる。
- 検証に「モジュール 0 件の生成結果が steering テンプレートと一字一句一致する」ケースを追加し、テンプレートと生成器の整合を恒久的に固定する。

## 理由

ゼロ状態の案内文は読者に有用であり、テンプレート側を生成器出力へ削る方向では失われる。
生成器に案内文を持たせれば、可読性を保ったまま greenfield の完全一致検査が成立する。
検証がテンプレートファイル自体と比較するため、将来どちらかを変更すると即座に検出される。

## 影響

この修正は B001（生成器と検証）と B003（テンプレート）にまたがる契約修正であり、メインセッションが直接実装した。
検証は RED（不一致の確認）から GREEN の順で追加した。
