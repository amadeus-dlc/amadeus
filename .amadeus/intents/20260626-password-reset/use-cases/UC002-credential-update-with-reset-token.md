# UC002：再設定トークンで認証情報を更新する

## 要求

- R002

## ストーリー

- S002

## アクター

- ACT001

## 外部システム

- なし

## 事前条件

- 利用者は再設定トークンを受け取っている。
- 利用者は新しい認証情報を入力できる。

## 基本フロー

1. 利用者は、再設定トークンと新しい認証情報を入力する。
2. システムは、再設定トークンが有効期限内で未使用であることを確認する。
3. システムは、再設定トークンに紐づくアカウントを確認する。
4. システムは、アカウントの認証情報を更新する。
5. システムは、再設定トークンを消費済みにする。
6. システムは、認証情報更新の完了を利用者へ返す。

## 代替フロー

- 再設定トークンが期限切れの場合、システムは認証情報を更新しない。
- 再設定トークンが使用済みの場合、システムは認証情報を更新しない。
- 再設定トークンが不正な場合、システムは認証情報を更新しない。

## 事後条件

- 有効な再設定トークンを使った場合、アカウントの認証情報が更新されている。
- 認証情報更新に使われた再設定トークンは再利用できない。
