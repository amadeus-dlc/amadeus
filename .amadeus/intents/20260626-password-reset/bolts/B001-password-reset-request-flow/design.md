# B001：パスワード再設定要求フロー設計

## 概要

B001 は、利用者からのパスワード再設定要求を受け付け、再設定トークンを発行し、再設定手段の送信を依頼する。
要求 R001 とユースケース UC001 を満たすための Bolt 内設計であり、`tasks.md` の入力として使う。

## 責務境界

所有するもの:

- パスワード再設定要求の入力受付。
- アカウント存在有無を推測できない応答。
- 再設定トークン発行条件。
- 再設定手段送信依頼の成立条件。

所有しないもの:

- 認証情報更新。
- 再設定トークンの消費。
- メール配信基盤の配送保証。
- アカウント存在有無を推測できる詳細理由の通知。

依存してよいもの:

- BC001 のアカウントモデル。
- 再設定手段送信先としての EXT001。
- R001、S001、UC001。

再確認が必要になる条件:

- 再設定手段がメール以外に増える。
- 再設定要求の入力識別子が複数になる。
- アカウント存在有無を推測できない応答の制約が変わる。

## 構成

- 要求入力を受け付ける境界。
- アカウント識別子を扱う処理。
- 再設定トークンを発行する処理。
- 再設定手段送信を依頼する処理。
- 利用者へ同一性のある応答を返す処理。

実装ファイルは未確定である。
Task では、責務単位を作業へ落とし、実装ファイルの確定は後続の Spec または Construction で扱う。

参照する要求:

- R001

- UC001

- PRE002

入出力:

- 入力: 利用者が提示するアカウント識別子。
- 出力: アカウント存在有無を推測できない再設定要求受付応答。
- 副作用: 条件を満たす場合、再設定トークンを発行し、再設定手段送信を依頼する。

## 検証方針

- アカウントに紐づく連絡先が存在する場合、再設定要求を受け付けることを確認する。
- 再設定トークンが期限付きで発行されることを確認する。
- 存在しないアカウントに対しても、存在有無を推測できる応答差が出ないことを確認する。
- 再設定手段送信依頼に失敗した場合でも、存在有無を推測できる情報を返さないことを確認する。

## Task への入力

- T001 は、利用者が入力するアカウント識別子と受付境界を定義する。
- T002 は、再設定トークン発行条件と再設定手段送信依頼を定義する。
- T003 は、アカウント存在有無を推測できない応答を検証する。
