# B002：認証情報更新フロー設計

## 概要

B002 は、有効な再設定トークンを検証し、対象アカウントの認証情報を更新し、再設定トークンを消費済みにする。
要求 R002 とユースケース UC002 を満たすための Bolt 内設計であり、`tasks.md` の入力として使う。

## 責務境界

所有するもの:

- 再設定トークンの有効性確認。
- 認証情報更新の成立条件。
- 再設定トークン消費と再利用不可の確認。
- 失敗時に認証情報を更新しない制約。

所有しないもの:

- 再設定トークンの発行。
- 再設定手段の送信。
- ログイン認証全体。
- アカウント停止や強制ログアウト。

依存してよいもの:

- B001/T002 で定義する再設定トークン発行の振る舞い。
- BC001 のアカウントモデル。
- R002、S002、UC002。

再確認が必要になる条件:

- 再設定トークンの検証方式が変わる。
- 認証情報更新後のセッション制御を同じ Intent に含める。
- 失敗時の通知や監査証跡が要求に追加される。

## 構成

- 再設定トークンを検証する処理。
- 認証情報更新要求を受け付ける境界。
- 認証情報を更新する処理。
- 再設定トークンを消費済みにする処理。
- 失敗時に認証情報を変更しない制御。

実装ファイルは未確定である。
Task では、責務単位を作業へ落とし、実装ファイルの確定は後続の Spec または Construction で扱う。

入出力:

- 入力: 再設定トークンと新しい認証情報。
- 出力: 認証情報更新の成否。
- 副作用: 更新成功時、対象再設定トークンを消費済みにする。

## 検証方針

- 有効な再設定トークンで認証情報を更新できることを確認する。
- 期限切れ、使用済み、不正な再設定トークンを拒否することを確認する。
- 認証情報更新後、対象再設定トークンが再利用できないことを確認する。
- 失敗時に認証情報が更新されないことを確認する。

## Task への入力

- T004 は、再設定トークン検証の条件を定義する。
- T005 は、認証情報更新と再設定トークン消費の整合を定義する。
- T006 は、消費後の再利用不可と失敗時の非更新を検証する。
