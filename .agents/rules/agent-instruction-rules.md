# Agent Instruction Rules

このルールは、このリポジトリでエージェント向けの指示、skill、運用ルール、レビュー方針を書くときに適用する。

## 基本方針

エージェントへの指示は、まず望ましい行動を肯定形で書く。

否定形の指示は、読み手の注意を禁止対象へ向ける。
そのため、未発生の失敗を予防する目的で禁止事項を列挙しすぎると、本来の目的に使う注意が削がれる。

## SHOULD

- 望ましい出力、判断順序、確認方法を先に書く。
- 制約を書く前に、通常時に取るべき行動を書く。
- 禁止事項は、既に起きた失敗、不可逆な操作、外部契約、安全上の制約に限定する。
- 禁止事項を書く場合は、代わりに取るべき行動を同じ場所に書く。
- 新しい禁止事項は、肯定形の指示だけでは防げなかった失敗を確認してから追加する。

## SHOULD NOT

- 起きていない失敗を予防しすぎて、禁止事項を列挙しない。
- 「しないこと」だけを書いて、期待する行動を曖昧にしない。
- 低リスクな表現ゆれや好みの違いを、禁止事項として固定しない。

## 例

望ましい書き方。

```md
作業前に対象ファイルを読み、既存の見出し、語彙、粒度に合わせる。
```

失敗後に追加する書き方。

```md
既存の見出しを確認せずに、新しい見出し体系を作らない。
見出し体系を変える必要がある場合は、先に理由を説明する。
```
