# bounded-contexts validation

## 対象

- `.amadeus/domain/bounded-contexts.md`

## 必須見出し

`bounded-contexts.md` は、次の見出しを持つ。

- `一覧`
- `外部境界`
- `コンテキスト間の依存`
- `パターン分類`

## 一覧表

`一覧` の表は、次の列を持つ。

- `識別子`
- `名前`
- `サブドメイン`
- `役割`
- `モデル`
- `契約`

`識別子` は `BCnnn` の形式にする。
同じ表の中で `識別子` を重複させない。

対象 Intent の `inception.gate` が `passed` の場合、Inception 以降の Unit は、`units.md` の `コンテキスト` から `.amadeus/domain/bounded-contexts.md` の BC を参照する。

`モデル` は `bounded-contexts/<bounded-context-id>/models.md` を参照する。
`契約` は `bounded-contexts/<bounded-context-id>/contracts.md` を参照する。

## 外部境界

`外部境界` の表は、次の列を持つ。

- `コンテキスト`
- `名前`
- `役割`
- `根拠`

`コンテキスト` には、`.amadeus/domain/bounded-contexts.md` の一覧に存在する境界づけられたコンテキスト ID を書く。
`名前` は ID 化しない。`traceability.md` の `ドメインモデルからの追跡` で `種別` が `境界` の行は、この `名前` に存在する値を参照する。
`役割` と `根拠` は空欄にしない。

## コンテキスト間の依存

`コンテキスト間の依存` の表は、次の列を持つ。

- `Downstream`
- `Upstream`
- `依存内容`
- `組織パターン`
- `統合パターン`
- `状態`

`Downstream` と `Upstream` には、`.amadeus/domain/bounded-contexts.md` の一覧に存在する境界づけられたコンテキスト ID、または `なし` を書く。

`依存内容` と `状態` は空欄にしない。

`Upstream` が `なし` の場合、`組織パターン` と `統合パターン` は `該当なし` にする。
この場合も、`状態` に理由を書く。

`Upstream` が `なし` でない場合、`組織パターン` は次のいずれかにする。

- `パートナーシップ`
- `別々の道`
- `順応者`
- `顧客／供給者`

`Upstream` が `なし` でない場合、`統合パターン` は次のいずれかにする。

- `共有カーネル`
- `巨大な泥団子`
- `公開ホストサービス（OHS）`
- `公表された言語（PL）`
- `腐敗防止層（ACL）`

## パターン分類

`パターン分類` は、組織パターン4種類と統合パターン5種類をすべて列挙する。

組織パターンは次の4種類である。

- `パートナーシップ`
- `別々の道`
- `順応者`
- `顧客／供給者`

統合パターンは次の5種類である。

- `共有カーネル`
- `巨大な泥団子`
- `公開ホストサービス（OHS）`
- `公表された言語（PL）`
- `腐敗防止層（ACL）`
