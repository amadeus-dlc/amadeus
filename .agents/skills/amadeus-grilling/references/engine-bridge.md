# Engine Bridge: AI-DLC Directive Questions to amadeus-grilling

This reference defines how an AI-DLC engine directive that requires filling a
`<stage>-questions.md` file connects to the `amadeus-grilling` interview
protocol. It is prompt-layer only: it does not modify engine code under
`.claude/tools/`, `.claude/aidlc-common/`, `.claude/sensors/`, or
`.claude/hooks/`.

## When this applies

Apply this bridge whenever an engine directive (a stage prompt driven by the
AI-DLC engine copied under `.claude/`) instructs the agent to fill a
`<stage>-questions.md` file with answers before the stage can proceed.

## Procedure

1. Present the directive's questions one at a time, following the
   `amadeus-grilling` protocol: each question includes a recommended answer
   and its rationale. Do not line up multiple questions at once.
2. Wait for the human's response before presenting the next question.
3. For each structured question:
   - Before presenting the question, run:
     ```sh
     bun .claude/tools/aidlc-log.ts decision --stage <slug> --decision "<summary>" --options "<csv>"
     ```
   - After the human responds, run:
     ```sh
     bun .claude/tools/aidlc-log.ts answer --stage <slug> --details "<exact choice>"
     ```
4. Write the confirmed answers into the `<stage>-questions.md` file using the
   upstream `[Answer]:` tag format. The questions file remains the source of
   truth for the stage's confirmed answers.
5. User-facing question text is Japanese. The questions file's structure,
   headings, and tag labels (including `[Answer]:`) stay in the upstream
   format.

## Non-goals

This bridge does not change the engine's stage protocol, question wording
generated by the engine, or the `<stage>-questions.md` file schema. It only
governs how the agent conducts the interview and logs it while filling that
file.
