# 日本語技術文書の文章規範

日本語で技術文書を書く、または推敲するときは、この規範に従う。

## 整形

- 一文ごとに改行する。
- 段落の区切りは空行で示す。
- コード、差分、ログ、設定ファイルの断片はコードブロックで示す。
- 初出で定義する用語は太字にする。
- すでに導入した語を話題として指すときは「」を使う。
- 日本語の地の文や見出しで、em ダッシュ、horizontal bar、二倍ダッシュを使わない。
- 日本語の並列に中黒を使わない。
- 用語と定義を並べる箇条書きは、全角コロンで「**用語**：説明」と書く。

## 段落と論証

- 一つの段落には一つのトピックだけを置く。
- 段落の最初の文で、その段落が何を扱うか分かるようにする。
- 前の段落との関係が必要な場合は、接続表現で明示する。
- 論証は一方向に進める。
- 否定するときは、否定する命題を具体的に書く。
- 因果を述べるときは、なぜそうなるのかを一文で示す。

## 読み手の負荷

- 後で参照しない固有名や実装詳細を出しすぎない。
- 抽象語の指す内容が曖昧な場合は、その場で具体化する。
- 新しい例や場面を追加するときは、前の例と何が違うのかを明示する。
- 確認していないことを、確認済みの事実のように書かない。

## 文体

- 「重要なのは」「正面から」「深掘りする」「多角的に」のように、中身のない前置きを避ける。
- 同じ主張を言い換えて繰り返さない。
- 過剰な強調や演出を避ける。
- 推量、仮定、未確認事項は断定しない。
- 根拠が本文内にあることは、弱めず具体的に書く。

## 用語

- このリポジトリのドメイン語彙は `CONTEXT.md` と `.amadeus/glossary.md` に合わせる。
- 同じ概念に別名を作らない。
- 未確定語を確定語彙として扱わない。
