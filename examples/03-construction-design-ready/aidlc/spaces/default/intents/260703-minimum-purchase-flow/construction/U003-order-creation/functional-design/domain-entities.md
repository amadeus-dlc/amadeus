# Domain Entities：注文作成

## 目的

注文作成が扱うドメインの概念と関係を定義する。

## Domain Entity

| Entity | 責務 | 主な属性 |
|---|---|---|
| 注文 | 購入者が作成した注文を表す。リレーショナルデータベースに記録された注文だけが注文の識別子を持つ | 注文の識別子、商品（商品識別子と、注文作成時点の商品名、価格の写し）、数量、金額、合計金額、作成日時 |

## 関係

注文は、商品選択（U001）から引き継いだ 1 件の商品識別子と数量から組み立てる（interaction-spec.md の商品の選択）。
注文が保持する商品名と価格は、注文作成時点の商品の値の写しであり、商品情報の後日の変更に影響されない。
注文の識別子は、注文記録がリレーショナルデータベースへの保存時に発行する（application-design の decisions.md の 4）。

## Domain Map と Context Map への反映候補

| 候補 | 種別 | 判断 |
|---|---|---|
| 商品選択と注文作成の間の語彙（商品識別子、商品名、価格） | 共有境界 | 見送り |

見送りの理由は次である。
Domain Map に採用済みの Subdomain と Bounded Context がなく、この語彙は本 Intent 内の領域間の受け渡しに留まるため、共有境界として採用する判断材料がない。
