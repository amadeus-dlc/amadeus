# Domain Entities：注文作成

## 目的

注文作成に必要な Domain Entity と関係を定義する。

## Domain Entity

| Entity | 責務 | 主な属性 |
|---|---|---|
| 注文 | 購入者の購入意思を記録する。 | 注文識別子、商品、数量、金額、合計金額、作成日時 |
| 注文内容 | 注文作成前に購入者が確認する内容を表す。 | 商品名、数量、金額、合計金額 |
| 注文可否 | 在庫参照結果に基づいて注文を作成できるかを表す。 | 状態、理由 |

## 関係

選択済み商品と数量から注文内容を作る。
注文可否が注文可能である場合だけ、注文内容から注文を作成し、注文記録へ渡す。
注文記録はリレーショナルデータベースで注文識別子を発行する。

## Domain Map と Context Map への反映候補

| 候補 | 種別 | 判断 |
|---|---|---|
| 注文作成 | 共有境界 | 未確認 |
| 注文作成が在庫参照結果を利用する依存 | コンテキスト間依存 | 未確認 |

## 未確認事項

- 注文作成を独立した Bounded Context として扱うかは未確認である。
- 在庫参照結果の最終的な属性は B002 で確定する。
