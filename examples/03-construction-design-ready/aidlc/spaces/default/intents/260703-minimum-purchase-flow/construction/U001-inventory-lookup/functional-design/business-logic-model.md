# Business Logic Model：在庫参照

## 目的

在庫管理システムの在庫情報を、購入フロー内の判断に使える在庫状況として提供するロジックを定義する。

## 対象 Unit

U001 在庫参照。
在庫管理システム（EXT001）の REST API から在庫情報を取得し、参照失敗を検出して呼び出し元へ伝える（R006、R007、R008）。

## 業務ロジック

在庫参照は、商品識別子の一覧を受け取り、在庫管理システムの REST API へ在庫参照の要求を送る。
応答を商品ごとの在庫状況へ変換して返す。

接続失敗、時間切れ、解釈できない応答は、いずれも参照失敗として検出し、呼び出し元へ伝える。
参照失敗時の振る舞いの判断（表示と注文可否）は行わず、商品一覧サービスと注文作成サービスに委ねる（GD001、GD006）。

在庫参照は在庫情報を読み取るだけであり、在庫の引き当てや更新は行わない。

## 入力

- 商品識別子の一覧（1 件以上）

## 出力

- 商品ごとの在庫状況（商品識別子、在庫の有無、在庫数量）
- または参照失敗（在庫状況が得られなかったという事実）

## 未確認事項

- 在庫管理システムの API 仕様（エンドポイント、認証方式、応答形式）と接続条件は未確認である。external-dependency-map.md の対応に従い、契約を仮置きしたスタブで進め、確定後に接続部分へ反映する。
- 応答に一部の商品しか含まれない場合の扱いは未確認である。API 仕様の確認時に確定する。
