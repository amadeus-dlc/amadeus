# Business Rules：在庫参照

## 目的

在庫参照で守る業務ルールと、商品選択および注文作成へ渡す契約を定義する。

## 業務ルール

| # | ルール | 根拠 |
|---|---|---|
| 1 | 商品一覧には商品ごとの在庫状況を表示する。 | R001、R005 |
| 2 | 在庫ありの商品だけを選択可能にする。 | R002、R005 |
| 3 | 在庫なしの商品は選択不可にする。 | R002、R005 |
| 4 | 在庫参照に失敗した場合は、在庫情報を取得できない旨を表示する。 | R006 |
| 5 | 在庫を確認できない商品では注文を作成しない。 | R006 |
| 6 | 在庫参照結果は注文可否として注文作成へ渡す。 | R002、R005、R006 |

## 例外

- 在庫管理システムの REST API 仕様が未確認のため、Code Generation 前に最小の API 契約を確定する。
- 在庫参照の一時的な失敗に対する再試行は未確認である。

## Intent Contracts

| 契約 | 事前条件 | 事後条件 |
|---|---|---|
| 在庫状況取得 | 商品識別子がある。 | 商品ごとの在庫状況が返る。 |
| 在庫状況表示 | 在庫参照結果がある。 | 購入者が商品ごとの在庫状況を確認できる。 |
| 選択可否判定 | 在庫参照結果がある。 | 在庫ありの商品だけが選択可能になる。 |
| 参照失敗時の縮退 | 在庫参照に失敗する。 | 在庫情報を取得できない旨が表示され、注文は作成されない。 |

## 未確認事項
