# Business Logic Model：在庫参照

## 目的

購入者が商品一覧で在庫状況を確認し、在庫参照の結果にもとづいて注文可否を判断できる状態を作る。

B002 では在庫管理システムとの REST API 連携、在庫状況表示、参照失敗時の縮退を扱う。
在庫参照結果は商品選択と注文作成の判断材料になる。

## 対象 Unit

U003 在庫参照

## 業務ロジック

在庫参照は、商品一覧に表示する商品識別子を入力として在庫管理システムの REST API を呼び出し、商品ごとの在庫状況を取得する。

商品一覧画面は、取得した在庫状況を商品ごとに表示する。
在庫ありの商品は選択可能とし、在庫なしの商品は選択不可として表示する。

在庫参照に失敗した場合は、在庫情報を取得できない旨を表示する。
在庫を確認できない商品では注文を作成できない。

在庫参照結果は注文可否として注文作成へ渡す。
注文作成は、在庫ありの場合だけ注文を記録できる。

## 入力

- 商品識別子
- 在庫管理システムの REST API 応答
- 在庫参照失敗

## 出力

- 商品ごとの在庫状況
- 商品ごとの選択可否
- 注文作成へ渡す注文可否
- 在庫情報を取得できない旨の表示

## 未確認事項

- 在庫管理システムの REST API 仕様は未確認である。
- 在庫状況の具体的な値の種類は未確認である。
- 在庫参照失敗時の再試行要否は未確認である。
