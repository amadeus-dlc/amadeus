# Components：260703-minimum-purchase-flow

greenfield のため既存コンポーネントはなく、すべて新規に設計する。
UI、サービス層、ドメインの 3 層で構成する（GD005）。

## 一覧

| コンポーネント | 責務 | 対応する要求 |
|---|---|---|
| 商品一覧画面 | 在庫状況つきの商品一覧の表示、数量指定の受け付け、商品選択、在庫参照失敗の表示 | R001, R002, R006, R008 |
| 注文内容確認画面 | 注文内容（商品、数量、金額、合計金額）の表示、注文作成の指示、在庫不足と在庫参照失敗の表示 | R003, R007, R008 |
| 注文完了画面 | 注文番号の提示 | R004 |
| 商品一覧サービス | 商品カタログと在庫参照を合成し、在庫状況つきの商品一覧を提供する | R001, R006 |
| 注文作成サービス | 在庫確認を経た注文の作成と記録を一連で行い、在庫不足と在庫参照失敗を判定する | R004, R005, R007, R008 |
| 商品カタログ | 販売対象の商品情報（商品識別子、商品名、価格）の提供 | R001 |
| 在庫参照 | 在庫管理システムの REST API から在庫情報を取得し、参照失敗を検出する | R006, R007, R008 |
| 注文 | 注文の組み立て、注文番号の採番、リレーショナルデータベースへの記録と注文番号による照会 | R004, R005 |

UI の 3 画面の相互作用の詳細は [interaction-spec.md](../refined-mockups/interaction-spec.md) に従う。
