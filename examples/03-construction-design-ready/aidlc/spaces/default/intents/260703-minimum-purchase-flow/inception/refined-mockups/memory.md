# Memory: refined-mockups

## Interpretations

- 精緻化の中心は例外系の確定に置いた。正常系の 3 画面構成は rough mockups から変えず、数量指定（GD004）、在庫参照失敗（GD001、R008）、在庫不足（R007）の表示と相互作用を追加した。
- 在庫参照失敗時は全商品を選択不可にした。部分的な在庫不明を許すより、GD001 の安全側の判断と整合するためである。

## Deviations

- 実行指示により逐次質問を行わず、指示内容と既存成果物から回答を確定した。質問と回答は refined-mockups-questions.md に記録した。

## Tradeoffs

- デザインシステムがないため、汎用 UI 部品の語彙で対応づけた。ライブラリ選定を先取りするより、Construction の技術スタック判断に委ねる方が手戻りが小さい。
- アクセシビリティはモック構成で満たせる観点だけを確認済みにし、配色やフォーカス表示など実装依存の観点は未確認のまま残した。

## Open questions

- コントラスト比とフォーカス表示の確認（UI コンポーネントライブラリの選定後）。
- 注文番号の形式（Construction の Functional Design で確定する）。
