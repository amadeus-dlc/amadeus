# Questions：Refined Mockups

実行指示により質問を対話で行わず、推奨回答を実行指示と既存成果物にもとづく判断で確定した。

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 詳細化で確定すべき相互作用はどれか | 在庫参照に失敗した場合の表示と、数量変更時の振る舞い | 推奨の通り確定した。R006 の縮退と R003 の数量指定が rough mockups で未確定だったため、interaction-spec.md と mockups.md で確定した |
| 2 | 使うデザインシステムや UI 規約はあるか | なしとして扱い、汎用の UI 部品へ対応付ける | 推奨の通り確定した。greenfield で UI 規約が存在せず、memory/ にも指定がないため、デザインシステムは未確認とし、汎用部品で対応付けた |
| 3 | アクセシビリティで満たすべき基準は何か | 基準の指定は未確認とし、モックの構造で確認できる観点だけを確認済みにする | 推奨の通り確定した。基準の指定がないため、視覚デザインと実装に依存する観点は未確認として Construction へ引き継ぐ |
