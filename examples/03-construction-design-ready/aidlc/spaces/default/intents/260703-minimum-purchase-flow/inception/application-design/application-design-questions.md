# Questions：Application Design

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 新しいコンポーネントの責務境界はどこか。 | UI、商品サービス、在庫参照、注文サービス、注文記録を分ける。 | ユーザー指示にある商品選択、注文作成、在庫参照を扱う新しいコンポーネントとサービス層の設計として採用した。 |
| 2 | サービスの分割単位はどうするか。 | 商品サービスと注文サービスに分け、在庫参照は商品サービスに含める。 | 最小購入フローの責務境界に従い、推奨回答を採用した。 |
| 3 | 既存アーキテクチャとの整合で譲れない点は何か。 | Greenfield のため既存コードベースはなく、TypeScript、Node.js、購入者向け Web UI、REST API、リレーショナルデータベースを制約にする。 | ユーザー指示と Ideation の制約に従い、推奨回答を採用した。 |
