# Accessibility Checklist：260703-minimum-purchase-flow

プロジェクトに明示されたアクセシビリティ基準はないため、WCAG 2.1 レベル AA を目安に基本観点を確認する。
基準の正式な確定は未確認のまま残し、Construction の NFR Requirements で扱う。

## 確認

| 観点 | 基準 | 状態 |
|---|---|---|
| キーボード操作 | 商品選択、数量入力、注文作成、再試行のすべての操作をキーボードだけで完了できる | 確認済み |
| フォームラベル | 数量入力フィールドに商品名と関連づいたラベルを付ける | 確認済み |
| エラー通知 | 在庫参照失敗と在庫不足のエラーをテキストで通知し、色だけに依存しない | 確認済み |
| 無効状態の伝達 | 選択不可のボタンは無効理由（在庫なし、在庫不明）をテキストで判別できる | 確認済み |
| コントラスト | テキストと背景のコントラスト比 4.5:1 以上 | 未確認 |
| 代替テキスト | 警告アイコンに代替テキストを付ける | 確認済み |
| フォーカス表示 | フォーカス位置を視覚的に判別できる | 未確認 |

「確認済み」はモックの構成として満たせることを確認した状態であり、実装の検証は Construction が扱う。
コントラストとフォーカス表示は、具体の配色とコンポーネントライブラリが未確定のため未確認である。
