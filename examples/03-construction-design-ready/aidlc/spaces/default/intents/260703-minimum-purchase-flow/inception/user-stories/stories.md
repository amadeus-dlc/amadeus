# Stories：260703-minimum-purchase-flow

## 一覧

| ID | ストーリー | 対応する要求 |
|---|---|---|
| S001 | 購入者として、商品一覧で商品名、価格、在庫状況を見たい。なぜなら在庫のある商品の中から注文対象を選びたいからだ。 | R001、R005 |
| S002 | 購入者として、在庫がある商品を選択したい。なぜなら注文できる商品だけに選ぶ時間を使いたいからだ。 | R002 |
| S003 | 購入者として、注文を作成する前に数量を指定して注文内容を確認したい。なぜなら作成後に訂正する手段がなく、間違いのない注文にしたいからだ。 | R003 |
| S004 | 購入者として、確認した内容で注文を作成し、注文の識別子を受け取りたい。なぜなら注文が受け付けられたことを確かめたいからだ。 | R004 |
| S005 | 購入者として、在庫状況を確認できない商品は選択できないと分かる形で案内されたい。なぜなら届くかどうか分からない注文を作りたくないからだ。 | R006 |
