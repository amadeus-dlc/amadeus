# Questions：Requirements Analysis

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 在庫参照に失敗した場合、注文作成をどう扱うか | 注文作成を進めず、エラーを表示して再試行を促す | 注文作成を進めず、在庫を確認できない旨を表示して再試行を促す（安全側に倒す。GD001） |
| 2 | 購入者の特定に認証は必要か | 最小フローでは認証を導入しない | 認証を導入しない。注文の確認は注文番号で行う（GD002） |
| 3 | 基本の決済を最小フローに含めるか | 含めない（注文の作成と記録まで） | 含めない。決済手段の拡張はバックログ Could のまま維持する（GD003） |
| 4 | 数量の指定方法はどうするか | 商品選択時に指定し、既定値 1、1 以上の整数 | 商品選択時に指定する。既定値 1、1 以上の整数だけを受け付ける（GD004） |
| 5 | 在庫管理システムの API 仕様は確認できたか | 未確認のまま振る舞いレベルで要求を定義する | 未確認。要求は振る舞いレベルで定義し、API 仕様の確定は Construction の設計ステージで扱う |

本ステージは人間への逐次質問を行わず、実行指示と既存成果物から回答を確定した。
確定判断は [grillings.md](../grillings.md) の G001 に記録した。
