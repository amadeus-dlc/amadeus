# Constraint Register：利用者が商品を選択して注文を作成できる最小の購入フローを実現する

## 交渉不能な制約

| 制約 | 種別 | 根拠 |
|---|---|---|
| TypeScript と Node.js の Web アプリケーションとして扱う。 | 既存アーキテクチャ | ユーザー指示の技術前提。 |
| 購入者向け Web UI を含める。 | 既存アーキテクチャ | ユーザー指示の技術前提。 |
| 在庫管理システムとは REST API で連携する。 | 既存アーキテクチャ | ユーザー指示の技術前提。 |
| 注文はリレーショナルデータベースに記録する。 | 既存アーキテクチャ | ユーザー指示の技術前提。 |
| 決済手段の拡張、会員ランク別価格、注文キャンセルは今回の対象外にする。 | 対象外 | ユーザー指示のスコープバックログ条件。 |
