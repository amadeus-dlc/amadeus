# 背景と前提

## 背景

- EC サイト最小購入フローは、利用者が商品を選択して注文を作成できる最小の購入体験を提供するための開発である。
- 社内の自社サイト向け開発であり、外部市場での位置づけや build-vs-buy の判断はない。

## 前提

- TypeScript と Node.js の Web アプリケーションとして構築する。
- 購入者向け Web UI を提供する。
- 在庫管理システムとは REST API で連携する。
- 注文はリレーショナルデータベースに記録する。
- 開発は単独開発者で行う。
