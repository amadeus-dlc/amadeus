# Mockups：最小購入フロー

Ideation の wireframes を、要求とストーリーに対応づけて精緻化した詳細モックである。
数量は 1 固定（D006）のため、数量の入力要素は置かない。

## 商品一覧画面

対応する要求とストーリー: R001、R002、S001、S002

```plantuml
@startsalt
{
  <b>商品一覧
  ..
  {#
    商品名 | 価格 | 在庫状況 | 操作
    商品 A | 1,000 円 | 在庫あり | [選択]
    商品 B | 2,500 円 | 在庫あり | [選択]
    商品 C | 800 円 | 在庫なし | 選択不可
  }
  ..
  在庫状況は在庫管理システムから参照した在庫情報を表示する
}
@endsalt
```

## 商品一覧画面（在庫参照不能時）

対応する要求とストーリー: R006、S006

在庫管理システムから在庫情報を参照できない場合の表示である。
在庫を確認できない旨を表示し、すべての商品を選択できないようにする。

```plantuml
@startsalt
{
  <b>商品一覧
  ..
  <color:red>在庫を確認できないため、現在注文を受け付けられません。
  ..
  {#
    商品名 | 価格 | 在庫状況 | 操作
    商品 A | 1,000 円 | 確認不可 | 選択不可
    商品 B | 2,500 円 | 確認不可 | 選択不可
    商品 C | 800 円 | 確認不可 | 選択不可
  }
}
@endsalt
```

## 注文内容の確認画面

対応する要求とストーリー: R003、S003

```plantuml
@startsalt
{
  <b>注文内容の確認
  ..
  {#
    商品名 | 数量 | 価格
    商品 A | 1 | 1,000 円
  }
  ..
  合計: 1,000 円
  ..
  [注文を作成する] | [商品一覧へ戻る]
}
@endsalt
```

## 注文内容の確認画面（注文を作成できない場合）

対応する要求とストーリー: R005、R006、S005、S006

注文作成時に在庫が不足する場合、または在庫情報を参照できない場合の表示である。
注文は作成されず、理由を表示する。

```plantuml
@startsalt
{
  <b>注文内容の確認
  ..
  <color:red>在庫が不足しているため、注文を作成できませんでした。
  ..
  {#
    商品名 | 数量 | 価格
    商品 A | 1 | 1,000 円
  }
  ..
  合計: 1,000 円
  ..
  [商品一覧へ戻る]
}
@endsalt
```

在庫情報を参照できない場合は、メッセージを「在庫を確認できないため、注文を作成できませんでした。」に置き換える。
