# Stories：最小購入フロー

## 一覧

| ID | ストーリー | 対応する要求 |
|---|---|---|
| S001 | 購入者として、商品一覧で商品名、価格、在庫状況を確認したい。なぜなら注文する商品を選ぶ判断材料が必要だからだ。 | R001 |
| S002 | 購入者として、在庫のある商品だけを選択したい。なぜなら注文できない商品を選んで手戻りすることを避けたいからだ。 | R002 |
| S003 | 購入者として、注文を作成する前に注文内容を確認したい。なぜなら意図しない内容で注文が作られることを避けたいからだ。 | R003 |
| S004 | 購入者として、注文を作成して注文の識別子を確認したい。なぜなら注文が受け付けられたことを確かめたいからだ。 | R004 |
| S005 | 販売管理者として、在庫が不足する商品の注文が作成されないようにしたい。なぜなら出荷できない注文への対応が発生するからだ。 | R005 |
| S006 | 販売管理者として、在庫を確認できない間は注文を受け付けないようにしたい。なぜなら判断材料のない注文は正しく作成された注文と言えないからだ。 | R006 |
