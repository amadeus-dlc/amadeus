# ナレッジ

## 背景

- EC サイト最小購入フローは、社内の自社サイト向け開発である。
- 外部市場での位置づけや build-vs-buy の判断はない。

## 前提

- 開発は単独開発者で行う。
- 在庫管理システムとの連携という統合制約がある。
- 購入者向けの UI は Web UI として提供する。
- 主要領域は商品選択、注文作成、在庫参照の 3 つである。
- TypeScript と Node.js の Web アプリケーションとして開発する。
- 在庫管理システムとは REST API で連携する。
- 注文はリレーショナルデータベースに記録する。

## 未確認事項

- 主目的の成功指標をどう測るかを確認する。
- Web UI のフレームワークとリレーショナルデータベースの具体的な製品を確認する。
- 開発標準（型運用、品質基準、テスト方針）と代表コマンドを確認する。
- プロジェクト構造の編成方針と命名規約を確認する。
- 販売管理者の具体的な役割と関心を確認する。
- 守るべき制約や禁止事項の有無を確認する。
