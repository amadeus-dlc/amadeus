# D002: 注文作成 Bolt 境界判断

## 背景

- ユーザー指定により、R004 注文作成は B001 の責務である。
- Traceability では B002 の要求に R004 を含めない必要がある。

## 判断

- R004 注文作成は B001 注文作成だけへ追跡する。
- B002 注文内容確認は、R001、R002、R003 だけへ追跡する。

## 理由

- 注文内容確認と注文作成は、顧客の流れでは連続するが、Construction での実施責務が異なるため。
- B002 に R004 を含めると、注文作成の責務が確認処理へ混ざるため。

## 影響

- `traceability.md` の `要求からの追跡` と `ボルトからの追跡` は、R004 と B001 の対応を双方向に一致させる。
- B001 は B002 に依存する。
