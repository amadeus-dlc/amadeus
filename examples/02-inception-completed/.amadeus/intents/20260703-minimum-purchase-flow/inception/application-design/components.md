# Components：最小購入フロー

greenfield の Intent であり、すべて新規コンポーネントである。
画面、サービス、外部連携、永続化の 4 つの責務に分ける。

## 一覧

| コンポーネント | 責務 | 対応する要求 |
|---|---|---|
| C001 商品一覧画面 | 商品名、価格、在庫状況の表示。数量の指定と商品選択の受け付け。在庫参照不能時の通知表示 | R001、R002、R006 |
| C002 注文内容確認画面 | 選択した商品、数量、価格、合計金額の表示。注文作成の指示と商品一覧への引き返し | R003 |
| C003 注文完了画面 | 発行された注文番号の表示 | R004 |
| C004 商品閲覧サービス | 商品情報と在庫状況を合成した商品一覧の提供。在庫参照不能時は在庫不明として通知 | R001、R005、R006 |
| C005 注文作成サービス | 注文の作成、注文番号の発行、注文の記録の指示 | R004 |
| C006 在庫参照クライアント | 在庫管理システム（EXT001）への REST API 呼び出しによる在庫情報の取得と、参照不能の検出 | R005、R006 |
| C007 商品リポジトリ | 登録済みの商品情報の参照 | R001 |
| C008 注文リポジトリ | 注文のリレーショナルデータベースへの記録と、注文番号による参照 | R004 |

商品情報の登録手段はこの Intent の対象外であり、未確認である。
C007 は登録済みの商品情報を読み取る前提で設計する。
