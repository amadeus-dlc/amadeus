# Requirements：最小購入フロー

## 一覧

| ID | 要求 | 由来する成功条件 |
|---|---|---|
| R001 | 購入者が商品一覧で商品名、価格、在庫状況を確認できる | 購入者が商品を選択して注文を作成できる。注文作成の判断に在庫管理システムの在庫情報を参照できる |
| R002 | 購入者が在庫のある商品を選択できる | 購入者が商品を選択して注文を作成できる |
| R003 | 購入者が注文内容を確認してから注文作成へ進める | 購入者が商品を選択して注文を作成できる |
| R004 | 購入者が注文を作成し、注文の識別子を確認できる | 購入者が商品を選択して注文を作成できる |
| R005 | 注文作成の判断に在庫管理システムの在庫情報を使う | 注文作成の判断に在庫管理システムの在庫情報を参照できる |
| R006 | 在庫情報を参照できない場合は注文作成を受け付けない | 注文作成の判断に在庫管理システムの在庫情報を参照できる |

## R001：商品一覧の表示

購入者は、商品一覧画面で商品ごとの商品名、価格、在庫状況を確認できる。
在庫状況は、在庫管理システムから参照した在庫情報に基づいて表示する。

受け入れ条件:

- 商品一覧画面に、商品ごとの商品名、価格、在庫状況が表示される。
- 表示される在庫状況が、在庫管理システムから参照した在庫情報と一致する。

## R002：商品選択

購入者は、商品一覧画面で在庫のある商品を選択できる。
在庫のない商品は選択できない。

受け入れ条件:

- 在庫のある商品を選択すると、注文内容の確認画面へ進める。
- 在庫のない商品は選択操作ができない。

## R003：注文内容の確認

購入者は、選択した商品の商品名、数量、価格、合計金額を注文作成の前に確認できる。
確認画面からは、注文の作成へ進むか、商品一覧画面へ戻るかを選べる。
数量は 1 固定にする。判断の理由は phase の decisions（D006）に記録した。

受け入れ条件:

- 注文内容の確認画面に、商品名、数量、価格、合計金額が表示される。
- 商品一覧画面へ戻ると、注文は作成されない。

## R004：注文作成

購入者は、注文内容の確認画面から注文を作成できる。
作成された注文には注文の識別子が発行され、注文完了画面に表示される。
作成された注文は記録として残り、後から参照できる。

受け入れ条件:

- 注文を作成すると、注文完了画面に注文の識別子が表示される。
- 作成された注文が記録として残り、注文の識別子で参照できる。

## R005：注文作成時の在庫参照

注文作成の判断材料として、作成時点の在庫情報を在庫管理システムから参照する。
在庫が不足する場合は注文を作成しない。

受け入れ条件:

- 在庫が足りる場合、注文が作成される。
- 在庫が不足する場合、注文は作成されず、在庫が不足している旨が購入者に表示される。

## R006：在庫参照不能時の振る舞い

在庫管理システムから在庫情報を参照できない場合、注文作成を受け付けない。
商品一覧画面では在庫を確認できない旨を表示し、商品を選択できないようにする。
判断の理由は phase の decisions（D005）に記録した。

受け入れ条件:

- 商品一覧画面の表示時に在庫情報を参照できない場合、在庫を確認できない旨が表示され、商品を選択できない。
- 注文作成時に在庫情報を参照できない場合、注文は作成されず、在庫を確認できない旨が購入者に表示される。
