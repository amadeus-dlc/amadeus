# Questions：Requirements Analysis

人間の指示が確認論点への回答を先に与えたため、grilling による対話は行っていない。
回答列には、指示と既存成果物から確定した内容を記録する。

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 在庫管理システムを参照できない場合の振る舞いはどうするか | 在庫を確認できない旨を表示し、注文作成を受け付けない | 注文作成を受け付けず、在庫を確認できない旨を購入者に表示する。R006 に反映し、理由を D005 に記録した |
| 2 | 注文の数量指定を最小スコープに含めるか | wireframes に数量の入力がないため、数量 1 固定にする | 数量は 1 固定にする。R003 に反映し、理由を D006 に記録した |
| 3 | 注文作成時に在庫をあらためて確認するか | 作成時点の在庫情報で判断する | 注文作成の判断材料として作成時点の在庫情報を参照し、不足する場合は注文を作成しない。R005 に反映した |
| 4 | 在庫管理システムの連携仕様の確認先はどこか | 未確認のまま残す | 未確認。要求は連携手段に依存しない表現で書き、連携手段の確定は Application Design の設計判断に委ねる |
| 5 | 受け入れ条件はどう観測するか | 購入者向け UI の画面操作と注文の記録で観測する | 購入者向け UI の画面操作と、作成された注文の記録で観測する |
