# Questions：Feasibility

人間の指示が確認論点への回答を先に与えたため、grilling による対話は行っていない。
回答列には、指示と既存成果物から確定した内容を記録する。

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 技術、運用、セキュリティ、依存のどこに不確実性があるか | 外部システム連携に不確実性がある | 技術スタックは TypeScript と Node.js で確定済みである。在庫管理システムとの REST API の具体的な連携仕様が未確認である |
| 2 | 交渉不能な制約は何か | 在庫情報の参照元を固定する | 在庫情報は在庫管理システムを参照元にし、REST API で連携する。注文はリレーショナルデータベースに記録する。決済手段の拡張、会員ランク別価格、注文キャンセルは対象にしない |
| 3 | 後続判断に効くリスク、前提、依存は何か | 外部依存と開発体制を登録する | 在庫管理システムへの依存、障害時の振る舞いの未定義、単独開発者という前提を RAID Log に登録した |
