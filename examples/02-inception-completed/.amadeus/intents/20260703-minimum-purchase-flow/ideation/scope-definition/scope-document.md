# Scope Document：最小購入フロー

## 最小スコープ

購入者が商品を選択して注文を作成できるまでの一連の流れである。

注文作成の判断材料として、在庫管理システムから参照した在庫情報を含む。
この一連の流れが成立した時点で、Intent の成功条件を観測できる。

## 対象

- 商品選択
- 注文作成
- 在庫管理システムを参照元にする在庫参照
- 購入者向けの Web UI

## 対象外

- 決済手段の拡張
- 会員ランク別価格
- 注文キャンセル

対象外の作業候補は、捨てずにスコープバックログへ記録する。

## 順序の方針

価値先行を選ぶ。

順序の指定がないため、商品選択から注文作成までの一連の流れを最初に成立させ、最小の購入体験という価値を早く観測できる順序を選んだ。

## 期限

| 対象 | 期限 | 根拠 |
|---|---|---|

期限の指定は現時点ではない。
