# Memory: user-stories

## Interpretations

- ペルソナは購入者の 1 つだけにした。販売管理者は役割と関心が未確認であり、本 Intent の対象外のためである。
- R007（在庫不足）と R008（参照失敗）は購入者から見える結果が異なるため、別ストーリー（S005、S006）に分けた。

## Deviations

- 新たな判断論点がなかったため、質問を行わず user-stories-questions.md は作成しなかった。境界の確定判断は requirements-analysis の G001 を引き継いだ。

## Tradeoffs

- R005（注文の永続化）は購入者に直接見えない要求だが、注文番号による確認という価値の裏付けとして S004 に含めた。独立したストーリーを立てるより、価値表現として自然だと判断した。

## Open questions
