# Units：260703-minimum-purchase-flow

Unit の境界戦略はドメイン別にし、glossary.md の 3 領域（商品選択、注文作成、在庫参照）に合わせた。
各 Unit は UI とサービス層を縦に貫き、技術レイヤーだけを根拠にした分割はしていない。

## 一覧

| ID | Unit | 責務 | 対応する要求 |
|---|---|---|---|
| U001 | 商品選択 | 商品一覧画面と商品一覧提供コンポーネントで、商品名と価格の一覧表示と、商品の選択操作を成立させる | R001、R002 |
| U002 | 在庫参照 | 在庫参照コンポーネントで在庫管理システムの REST API から在庫状況を取得し、商品一覧画面の在庫状況表示と、参照失敗時の縮退（選択不可と案内表示）を成立させる | R001、R005、R006 |
| U003 | 注文作成 | 注文内容確認画面、注文完了画面と注文サービス（注文作成、注文記録）で、数量指定から注文のリレーショナルデータベース記録と注文の識別子表示までを成立させる | R003、R004 |
