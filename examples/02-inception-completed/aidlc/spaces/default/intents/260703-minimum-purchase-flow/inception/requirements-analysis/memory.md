## Tradeoffs

- 在庫参照に失敗した場合に注文を止める判断は、可用性より誤発注の防止を優先する。注文キャンセルが対象外である間はこの優先を維持し、キャンセルが実現したら再評価する。
- 認証と決済を範囲外にしたため、注文と購入者の紐付けはできない。最小購入フローの成立を優先した。
