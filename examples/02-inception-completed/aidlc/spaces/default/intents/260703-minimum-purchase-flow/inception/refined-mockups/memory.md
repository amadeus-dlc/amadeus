# Memory: refined-mockups

## Interpretations

- rough mockups の 3 画面（商品一覧、注文内容確認、注文完了）を維持し、画面の追加はせず要求とストーリーへの対応付けと相互作用の確定だけを行った。
- 在庫参照に失敗した場合の表示（R006）を、商品一覧画面の別状態として明示した。

## Deviations

- 実行指示により質問を対話で行わず、refined-mockups-questions.md に推奨回答と確定理由を記録する形へ代替した。

## Tradeoffs

- 在庫参照に失敗した場合、全商品を選択不可にする表示へ倒した。商品単位で参照失敗を切り分ける表示は、在庫管理システムの API 仕様が未確認の間は確定できないためである。
- デザインシステムを導入せず汎用の UI 部品へ対応付けた。最小購入フローの成立を優先し、UI 規約の整備は必要になった時点で扱う。
