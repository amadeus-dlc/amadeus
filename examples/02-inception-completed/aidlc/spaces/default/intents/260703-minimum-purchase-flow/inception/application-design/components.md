# Components：260703-minimum-purchase-flow

## 一覧

| コンポーネント | 責務 | 対応する要求 |
|---|---|---|
| 商品一覧画面 | 商品名、価格、在庫状況の一覧を表示し、在庫がある商品の選択操作を提供する。在庫を確認できない場合は案内を表示して選択を止める | R001、R002、R006 |
| 注文内容確認画面 | 選択した商品の数量指定と注文内容の確認を提供し、注文作成の操作を受け付ける | R003、R004 |
| 注文完了画面 | 作成した注文の識別子を表示する | R004 |
| 商品一覧提供 | 注文対象になる商品（商品名、価格）の一覧を提供する | R001 |
| 在庫参照 | 在庫管理システムの REST API から在庫情報を取得し、商品ごとの在庫状況（あり、なし、確認不可）へ変換する。参照に失敗した場合は確認不可として扱う | R005、R006 |
| 注文作成 | 数量を検証し、選択した商品から注文を組み立てて注文記録へ渡す | R003、R004 |
| 注文記録 | 注文をリレーショナルデータベースに記録し、注文の識別子を発行する | R004 |

UI の 3 画面は refined-mockups の 3 画面と 1 対 1 で対応する。
画面の相互作用の詳細は inception/refined-mockups/interaction-spec.md が定義する。
