# Requirements：260703-minimum-purchase-flow

## 一覧

| ID | 要求 | 由来する成功条件 |
|---|---|---|
| R001 | 商品一覧の表示 | 購入者が購入対象の商品を選択できる。 |
| R002 | 在庫状況にもとづく商品選択 | 購入者が在庫管理システムから得た在庫状態を購入判断に使える。 |
| R003 | 数量の指定と注文内容の確認 | 購入者が注文を作成できる。 |
| R004 | 注文の作成と記録 | 購入者が注文を作成し、注文がリレーショナルデータベースに記録される。 |
| R005 | 在庫情報の参照 | 購入者が在庫管理システムから得た在庫状態を購入判断に使える。 |
| R006 | 在庫参照に失敗した場合の縮退 | 購入者が在庫管理システムから得た在庫状態を購入判断に使える。 |
| R007 | 販売管理者が確認できる注文記録 | 販売管理者が作成された注文を確認できる状態になる。 |

## R001：商品一覧の表示

購入者が購入者向け Web UI の商品一覧画面で、注文対象の商品を閲覧できる。
商品ごとに商品名、価格、在庫状況を表示する。

受け入れ条件:

- 商品一覧画面を開くと、商品ごとに商品名、価格、在庫状況が表示される。
- 在庫状況は、在庫管理システムから参照した在庫情報を反映する。
- 商品一覧に表示する商品の取得元は未確認である。

## R002：在庫状況にもとづく商品選択

購入者は、在庫がある商品だけを選択できる。
在庫がない商品は選択不可として表示する。

受け入れ条件:

- 在庫がある商品には選択操作が表示され、選択すると注文内容の確認へ進める。
- 在庫がない商品は選択不可として表示され、選択操作ができない。
- 在庫あり、在庫なし、確認不可の表示文言は購入者が判別できる形で表示される。

## R003：数量の指定と注文内容の確認

購入者は、選択した商品の数量を指定し、注文を作成する前に注文内容を確認できる。
数量は 1 以上の整数で指定し、既定値は 1 にする。
注文キャンセルは対象外であるため、作成前の確認を必須にする。

受け入れ条件:

- 注文内容確認画面に、選択した商品の商品名、数量、金額、合計金額が表示される。
- 数量を 1 以上の整数へ変更でき、金額と合計金額が数量に応じて更新される。
- 0 以下の数量と整数以外の数量は指定できない。

## R004：注文の作成と記録

購入者は、確認した注文内容から注文を作成できる。
作成した注文はリレーショナルデータベースに記録され、注文の識別子が発行される。
本 Intent では決済手段の拡張を扱わないため、注文作成は決済なしで完了する。

受け入れ条件:

- 注文内容確認画面の注文作成操作で注文が作成され、注文完了画面に注文の識別子が表示される。
- 作成した注文が、商品、数量、金額、合計金額、作成日時とともにリレーショナルデータベースに記録される。
- 決済操作なしで注文作成が完了する。

## R005：在庫情報の参照

商品一覧の表示時に、在庫管理システムの REST API から商品の在庫情報を参照する。
API の具体的な仕様と接続条件は未確認であり、Delivery Planning の外部依存整理で確認先を特定し、Construction の設計までに確定する。

受け入れ条件:

- 商品一覧を開くと、在庫管理システムの REST API へ在庫参照の要求が送られる。
- 参照した在庫情報が、商品一覧の在庫状況の表示に反映される。
- API 仕様、認証方式、タイムアウト値は未確認として扱い、Construction の設計で確定する。

## R006：在庫参照に失敗した場合の縮退

在庫管理システムから在庫情報を参照できない場合、対象商品を選択不可として扱い、注文を作成しない。
注文キャンセルが対象外であり、在庫不明のまま受けた注文を回復する手段がないため、安全側に倒す。

受け入れ条件:

- 在庫管理システムが応答しない場合、またはエラーを返した場合、対象商品は選択不可として表示される。
- 在庫情報を参照できない旨が購入者に表示される。
- 在庫状況を確認できない商品では注文を作成できない。

## R007：販売管理者が確認できる注文記録

販売管理者が作成された注文を業務上確認できるように、注文記録は確認に必要な最小項目を保持する。
販売管理者向け UI は未確認であり、本 Intent では注文記録の成立を確認可能な状態にする。

受け入れ条件:

- 注文記録には、注文の識別子、商品、数量、金額、合計金額、作成日時が含まれる。
- 販売管理者向け UI の有無は未確認であり、Construction では注文記録の参照方法を確定する。
