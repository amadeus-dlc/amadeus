# Initiative Brief：利用者が商品を選択して注文を作成できる最小の購入フローを実現する

## 目的と成功条件

購入者が商品を選択し、在庫を参照したうえで注文を作成できる最小の購入体験を実現する。
成功条件は、商品選択、在庫参照、注文作成、注文のリレーショナルデータベース記録が完了できることである。

## スコープ境界

対象: 商品選択、在庫参照、注文作成、注文記録、販売管理者が注文を確認できる状態。

対象外: 決済手段の拡張、会員ランク別価格、注文キャンセル。

## バックログ要約

| 項目 | 優先度 |
|---|---|
| 決済手段の拡張 | Should |
| 注文キャンセル | Should |
| 会員ランク別価格 | Could |

## 制約

- TypeScript と Node.js の Web アプリケーションとして扱う。
- 購入者向け Web UI を含める。
- 在庫管理システムとは REST API で連携する。
- 注文はリレーショナルデータベースに記録する。

## 体制

開発は単独開発者で行う。
そのため、Team Formation は skip した。

## モック

`ideation/rough-mockups/wireframes.md` と `ideation/rough-mockups/user-flow.md` に、商品選択から注文作成までの粗い UI とフローを記録した。

## Inception への引き継ぎ

Inception では、在庫管理システムの API 仕様、在庫不足時の注文可否、注文記録項目、購入者向け UI の詳細を確認する。
