## コア能力

- 商品選択。
- 注文作成。
- 在庫参照。

## 主要ユースケース

- 購入者が商品を選択し、在庫を確認して注文を作成する。
- 販売管理者が作成された注文を確認する。

## 価値仮説

- 最小の購入フローを提供すると、購入者は問い合わせや手作業を介さずに購入意思を記録できる。

## アーキテクチャ

- TypeScript と Node.js による Web アプリケーションとして構成する。
- 購入者向け Web UI を提供する。
- 在庫管理システムとは REST API で連携する。
- 注文はリレーショナルデータベースに記録する。

## 開発標準

### 型安全性

- TypeScript の型を使い、注文作成と在庫参照の入出力を明示する。

### 品質基準

- 購入者が商品選択から注文作成まで到達できることを確認する。
- 在庫管理システムの応答に依存する判断は、失敗時の扱いを後続 stage で明確にする。

### テスト

- 最小購入フローの正常系を検証対象にする。
- 在庫参照の異常系は後続 stage で詳細化する。
