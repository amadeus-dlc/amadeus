# D003：順序の方針を価値先行にする

## 背景

Scope Definition では、作業の順序の方針（リスク先行、価値先行、依存先行）を確認する。
人間の指示に順序の指定はなかった。

## 判断

順序の方針を価値先行にする。

## 理由

商品選択から注文作成までの一連の流れを最初に成立させると、最小の購入体験という価値を最も早く観測できる。
統合制約（在庫参照）はこの流れの内側にあり、リスク先行と価値先行で着手対象が変わらない。

## 影響

Inception の Delivery Planning は、walking skeleton に商品選択から注文作成までの流れを含める前提で順序付けを行う。
