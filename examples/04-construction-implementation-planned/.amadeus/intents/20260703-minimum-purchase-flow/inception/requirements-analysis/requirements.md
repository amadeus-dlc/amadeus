# Requirements：最小購入フロー

## 一覧

| ID | 要求 | 由来する成功条件 |
|---|---|---|
| R001 | 商品一覧の表示 | 購入者が商品を選択して注文を作成できる |
| R002 | 商品の選択と数量の指定 | 購入者が商品を選択して注文を作成できる |
| R003 | 注文内容の確認 | 購入者が商品を選択して注文を作成できる |
| R004 | 注文の作成と記録 | 購入者が商品を選択して注文を作成できる |
| R005 | 在庫情報の参照 | 注文作成の判断に在庫管理システムの在庫情報を参照できる |
| R006 | 在庫情報を参照できない場合の振る舞い | 注文作成の判断に在庫管理システムの在庫情報を参照できる |

## R001：商品一覧の表示

購入者が、販売対象の商品を一覧で確認できる。
一覧には、商品ごとに商品名、価格、在庫状況を表示する。
在庫状況は、在庫管理システムから参照した在庫情報を反映する。

受け入れ条件:

- 商品一覧画面に、商品ごとの商品名、価格、在庫状況が表示される。
- 表示される在庫状況が、在庫管理システムから参照した在庫情報と一致する。

## R002：商品の選択と数量の指定

購入者が、在庫のある商品を注文対象として選択し、数量を指定できる。
在庫のない商品は選択できない。

受け入れ条件:

- 在庫のある商品を選択すると、注文内容の確認へ進める。
- 在庫のない商品は選択できない。
- 数量を 1 以上の整数で指定できる。

## R003：注文内容の確認

購入者が、注文を作成する前に、選択した商品の内容を確認できる。
確認できる内容は、商品名、数量、価格、合計金額である。

受け入れ条件:

- 選択した商品の商品名、数量、価格と、合計金額が表示される。
- 注文を作成せずに商品一覧へ戻れる。

## R004：注文の作成と記録

購入者が、注文内容の確認から注文を作成できる。
作成した注文は一意の注文番号を持ち、記録される。
注文作成は購入の申し込みの記録までとし、決済を含めない（GD002）。

受け入れ条件:

- 注文を作成すると、一意の注文番号が発行される。
- 注文完了画面に注文番号が表示される。
- 作成した注文が記録され、注文番号で参照できる。

## R005：在庫情報の参照

商品の在庫状況は、在庫管理システムの在庫情報を参照元にする。
商品を選択できるかどうかの判断に、参照した在庫情報を使う。

受け入れ条件:

- 商品一覧の在庫状況が、在庫管理システムから参照した在庫情報を参照元にしている。
- 在庫のない商品では、注文内容の確認へ進めない。

## R006：在庫情報を参照できない場合の振る舞い

在庫管理システムから在庫情報を参照できない場合、対象商品の在庫状況を在庫不明として扱い、注文作成を受け付けない（GD001）。
購入者には、在庫情報を参照できないことを表示する。

受け入れ条件:

- 在庫情報を参照できない場合、対象商品を選択できない。
- 在庫情報を参照できないことが購入者に表示される。

## 未確認事項

- 在庫管理システムの REST API の具体的な連携仕様と確認先は未確認である。
  要求は、商品ごとの在庫状況を参照できる前提で定義した。
  連携仕様の確定は、Delivery Planning の外部依存として管理する。
