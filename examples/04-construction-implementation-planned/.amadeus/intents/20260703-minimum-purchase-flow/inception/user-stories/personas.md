# Personas：最小購入フロー

## 一覧

| ペルソナ | 特徴 | 主な目的 |
|---|---|---|
| 購入者 | 商品を選択して注文を作成する利用者（ACT001）。欲しい商品を最小の手順で注文できることに関心を持つ | 商品を選択して注文を作成する |
| 販売管理者 | 商品の販売と注文の状況を管理する担当者（ACT002）。steering のアクター定義では状態が未確認であり、この Intent では画面を操作しない | 注文が正しく作成されることを確認する |

購入者向け Web UI を操作するペルソナは購入者だけである。
販売管理者は、記録された注文の確認者として要求 R004 の受け入れ条件に間接的に関わる。
