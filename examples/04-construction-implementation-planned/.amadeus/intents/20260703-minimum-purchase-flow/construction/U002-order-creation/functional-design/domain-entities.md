# Domain Entities：注文作成

## 目的

注文作成が扱うドメインエンティティと関係を確定し、Code Generation のデータモデルの入力にする。

## Domain Entity

| Entity | 責務 | 主な属性 |
|---|---|---|
| 注文 | 購入の申し込みの記録。一意の注文番号の発行と、注文番号による参照 | 注文番号、商品の識別子、商品名、数量、価格、合計金額 |
| 注文内容 | 注文を作成する前に購入者が確認する内容の保持 | 商品名、数量、価格、合計金額 |

## 関係

- 注文は、選択した 1 つの商品に対する購入の申し込みであり、1 以上の整数の数量を持つ（glossary.md、R002、R003）。
- 注文内容は注文の作成前の確認に使う値であり、注文の作成によって注文として記録される。
- 注文の対象になる商品と数量は、商品閲覧（U001）の選択から画面遷移で受け渡される（unit-dependencies.md）。
