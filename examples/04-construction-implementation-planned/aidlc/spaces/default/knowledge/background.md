# 背景と前提

## 背景

- EC サイトの最小購入フローを examples 用 workspace として作成する。
- 社内の自社サイト向け開発であり、外部市場での位置づけや build-vs-buy の判断はない。

## 前提

- 利用者が商品を選択して注文を作成できることを主目的にする。
- 主なアクターは購入者と販売管理者である。
- 在庫管理システムとは REST API で連携する。
- 開発は単独開発者で行う。
- 購入者向けの UI がある。

## 未確認事項

- 商品一覧、商品詳細、注文確認のどこまでを最小 UI に含めるかは Inception で確認する。
- 在庫不足時の購入者への表示と注文可否は Inception で確認する。
