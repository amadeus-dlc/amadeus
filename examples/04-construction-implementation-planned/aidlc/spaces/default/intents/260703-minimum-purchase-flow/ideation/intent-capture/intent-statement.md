# Intent Statement：260703-minimum-purchase-flow

## 目的

利用者が商品を選択して注文を作成できる最小の購入フローを実現する。

## 対象

- **購入者**：EC サイト最小購入フローの主対象である。現状は商品を選んで注文を作成する手段がない。
- **販売管理者**：主なアクターとして挙げられているが、役割と関心は未確認である。

## 成功条件

- 購入者が Web UI で商品を選択できる。
- 購入者が選択した商品から注文を作成できる。
- 作成した注文がリレーショナルデータベースに記録される。
- 注文作成の過程で、在庫管理システムの在庫情報を REST API 経由で参照できる。

## 契機

対象 workspace は greenfield であり、購入フローがまだ存在しない。
社内の自社サイト向け開発として、最小の購入体験を最初に成立させる必要があるため、この Intent を開始する。

## 範囲

含めるもの:

- 商品選択
- 注文作成
- 在庫参照（在庫管理システムとの REST API 連携）

含めないもの:

- 決済手段の拡張
- 会員ランク別価格
- 注文キャンセル

含めないものは今回やらない作業候補であり、優先度付けは Scope Definition のスコープバックログで整理する。
