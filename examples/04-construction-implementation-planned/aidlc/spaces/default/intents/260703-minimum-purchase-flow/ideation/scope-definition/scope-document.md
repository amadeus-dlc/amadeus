# Scope Document：260703-minimum-purchase-flow

## 最小スコープ

購入者が Web UI で商品を選択し、在庫管理システムの在庫情報を参照した上で注文を作成でき、その注文がリレーショナルデータベースに記録されるまでを最小スコープとする。
この範囲で「利用者が商品を選択して注文を作成できる」という成功条件を観測できる。

## 対象

- 商品選択（購入者が注文対象の商品を選べること）
- 注文作成（選択した商品から注文を作成し、リレーショナルデータベースに記録すること）
- 在庫参照（在庫管理システムの REST API から在庫情報を参照すること）

## 対象外

- 決済手段の拡張
- 会員ランク別価格
- 注文キャンセル
- 販売管理者向けの機能（役割と関心が未確認のため、確認後に扱いを判断する）

対象外の作業候補は intent-backlog.md に優先度付きで記録する。

## 順序の方針

リスク先行を基本にする。
統合制約である在庫参照（在庫管理システムとの REST API 連携）を早い段階で通し、商品選択から注文作成までの最小経路を先に成立させる。
在庫管理システムの API 仕様が未確認であり、後回しにすると手戻りが最も大きい部分だからである。
