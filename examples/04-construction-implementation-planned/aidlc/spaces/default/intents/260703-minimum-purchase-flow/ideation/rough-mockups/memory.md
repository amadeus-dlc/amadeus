# Memory: rough-mockups

## Interpretations

- 画面は最小スコープに合わせて商品一覧、注文内容確認、注文完了の 3 つに絞った。
- 在庫なしの商品は選択不可として表示する解釈を採用した。確定は要求分析が扱う。

## Deviations

- 実行指示により逐次質問を行わず、指示内容と既存成果物から回答を確定した。質問と回答は rough-mockups-questions.md に記録した。

## Tradeoffs

- 数量指定やカート相当の中間状態を画面として起こさず、注文内容確認画面に含めた。最小の購入体験を確認できる粒度を優先した。

## Open questions

- 在庫参照に失敗した場合、商品一覧をどう表示するか。
- 数量の指定はどの画面で行うか。
