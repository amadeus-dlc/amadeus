# RAID Log：利用者が商品を選択して注文を作成できる最小の購入フローを実現する

## 記録

| 種別 | 内容 | 影響 | 対応 |
|---|---|---|---|
| 依存 | 在庫管理システムの REST API に在庫参照が依存する。 | API 仕様や障害時の扱いが購入フローに影響する。 | Inception で API 仕様とエラー時の要求を確認する。 |
| リスク | 在庫不足時の注文可否が未確認である。 | 購入者の操作結果と注文記録の整合性に影響する。 | Requirements Analysis で業務ルールを確定する。 |
| 前提 | 注文はリレーショナルデータベースに記録する。 | 注文作成の成功条件に永続化が含まれる。 | Requirements Analysis と Application Design で記録項目を具体化する。 |
| 課題 | 購入者向け UI の最小画面構成が未確認である。 | UI の範囲が広がる可能性がある。 | Rough Mockups で暫定フローを示し、Refined Mockups で詳細化する。 |
