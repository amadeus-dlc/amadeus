# RAID Log：260703-minimum-purchase-flow

## 記録

| 種別 | 内容 | 影響 | 対応 |
|---|---|---|---|
| 依存 | 在庫参照を在庫管理システムの REST API に依存する | API 仕様が未確認のまま設計すると手戻りが生じる | Inception の要求分析までに API 仕様と接続条件を確認する |
| リスク | 在庫管理システムが応答しない場合の注文作成の可否が未定義である | 注文作成の成立条件が揺れ、要求と設計が定まらない | 要求分析で在庫参照に失敗した場合の振る舞いを確定する |
| 前提 | 単独開発者で開発する | レビューと承認が開発者自身に集中する | ステージゲートで成果物を確認しながら進める |
| 課題 | 購入者の特定方法（認証の要否）が未確認である | セキュリティ要求と UI 設計に影響する | Inception の要求分析で確認する |
