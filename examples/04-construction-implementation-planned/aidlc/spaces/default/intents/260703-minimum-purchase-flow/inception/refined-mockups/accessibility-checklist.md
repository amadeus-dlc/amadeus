# Accessibility Checklist：260703-minimum-purchase-flow

満たすべきアクセシビリティ基準の指定は未確認である。
このチェックリストでは、モックの構造で確認できる観点を確認済みにし、視覚デザインと実装に依存する観点を未確認として Construction へ引き継ぐ。

## 確認

| 観点 | 基準 | 状態 |
|---|---|---|
| 選択不可の判別 | 選択不可の商品を、色だけでなく文言（選択不可、確認不可）で判別できる | 確認済み |
| エラーの通知 | 在庫参照の失敗と数量の入力エラーを、状態変化だけでなく文言で通知する | 確認済み |
| 操作の一貫性 | 画面間の遷移操作（選択、戻る、注文を作成する）がボタンとして一貫している | 確認済み |
| フォームのラベル | 数量入力フィールドに数量であることを示すラベルを付ける | 確認済み |
| キーボード操作 | すべての操作をキーボードだけで実行できる | 未確認 |
| コントラスト比 | 文字と背景のコントラスト比が読み取りに十分である | 未確認 |
| 支援技術への対応 | スクリーンリーダーが表とエラー文言を読み上げられる | 未確認 |

未確認の観点は、実装時の確認対象として Construction の設計と実装で扱う。
