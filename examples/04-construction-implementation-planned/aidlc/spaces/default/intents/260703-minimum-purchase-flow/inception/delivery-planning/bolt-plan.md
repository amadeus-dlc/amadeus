## B001：注文作成貫通（walking skeleton）

購入者が商品一覧から在庫のある商品を選択し、在庫参照を経て注文を作成し、注文番号を確認できるまでの経路を、UI からリレーショナルデータベースまで貫通させる（GD011）。
Bolt 内の Unit は依存 DAG に従い、U001（在庫参照）、U002（商品選択）、U003（注文作成）の順で実施する。

Definition of Done:

- 購入者が Web UI で在庫状況つきの商品一覧から商品と数量を選択できる（R001、R002、R006）。
- 購入者が注文内容を確認し、注文を作成して注文番号を確認できる（R003、R004）。
- 作成した注文がリレーショナルデータベースに記録され、注文番号で照会できる（R005）。
- 在庫不足と在庫参照失敗の例外系が要求どおりに振る舞う（R007、R008）。
- Build and Test（Stage 3.6）でビルドとテストが成功している。
- walking skeleton ゲートとして、Bolt PR を人間が承認して merge している。

confidence hypothesis: TypeScript と Node.js の Web アプリケーション、在庫管理システムとの REST API 連携、リレーショナルデータベースへの記録という統合前提が、一気通貫で成立することを証明する。この Bolt の出荷により、本 Intent の成功条件のすべてが観測可能になり、後続のスコープバックログ項目（注文キャンセルなど）を載せる土台が確定する。
