# Stories：260703-minimum-purchase-flow

## 一覧

| ID | ストーリー | 対応する要求 |
|---|---|---|
| S001 | 購入者として、販売中の商品の一覧を価格と在庫状況つきで見たい。なぜなら在庫のある商品の中から注文する商品を選びたいからだ。 | R001, R006 |
| S002 | 購入者として、在庫のある商品を数量を指定して選択したい。なぜなら必要な数だけ注文したいからだ。 | R002 |
| S003 | 購入者として、注文を作成する前に注文内容と合計金額を確認したい。なぜなら誤った内容で注文を作成したくないからだ。 | R003 |
| S004 | 購入者として、確認した内容で注文を作成して注文番号を受け取りたい。なぜなら注文が受け付けられたことを後から確認したいからだ。 | R004, R005 |
| S005 | 購入者として、注文しようとした商品の在庫が足りない場合は注文を作らずにその旨を知りたい。なぜなら届かない注文を作りたくないからだ。 | R007 |
| S006 | 購入者として、在庫が確認できない場合はその旨を知って再試行したい。なぜなら注文できるか分からないまま操作を進めたくないからだ。 | R008 |
