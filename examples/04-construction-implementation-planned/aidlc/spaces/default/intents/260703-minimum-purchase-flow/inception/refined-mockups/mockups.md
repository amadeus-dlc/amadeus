# Mockups：260703-minimum-purchase-flow

## 商品一覧画面

対応する要求とストーリー: R001、R002、R005、R006、S001、S002、S005

商品ごとに商品名、価格、在庫状況を表示する。
在庫がある商品だけに選択操作を表示し、在庫がない商品は選択不可として表示する。
在庫管理システムから在庫情報を参照できない場合は、参照できない旨の案内を表示し、対象商品を選択不可にする。

```plantuml
@startsalt
{
  {* EC サイト最小購入フロー | 商品一覧 }
  ..
  {#
    商品名 | 価格 | 在庫 | 操作
    商品 A | 1,000 円 | あり | [ 選択 ]
    商品 B | 2,000 円 | あり | [ 選択 ]
    商品 C | 3,000 円 | なし | 選択不可
  }
}
@endsalt
```

在庫情報を参照できない場合の表示は次にする。

```plantuml
@startsalt
{
  {* EC サイト最小購入フロー | 商品一覧 }
  ..
  { <&warning> 在庫情報を取得できません。在庫を確認できない商品は選択できません。 }
  ..
  {#
    商品名 | 価格 | 在庫 | 操作
    商品 A | 1,000 円 | 確認不可 | 選択不可
    商品 B | 2,000 円 | 確認不可 | 選択不可
    商品 C | 3,000 円 | 確認不可 | 選択不可
  }
}
@endsalt
```

## 注文内容確認画面

対応する要求とストーリー: R003、S003

選択した商品の商品名、数量、金額、合計金額を表示する。
数量は 1 以上の整数を指定でき、既定値は 1 にする。
数量を変更すると、金額と合計金額が数量に応じて更新される。

```plantuml
@startsalt
{
  {* EC サイト最小購入フロー | 注文内容の確認 }
  ..
  {#
    商品名 | 数量 | 金額
    商品 A | "1  " | 1,000 円
  }
  ..
  { 合計 | 1,000 円 }
  ..
  { 数量は 1 以上の整数で指定してください。 }
  ..
  { [ 商品一覧へ戻る ] | [ 注文を作成する ] }
}
@endsalt
```

## 注文完了画面

対応する要求とストーリー: R004、S004

作成した注文の識別子を表示し、注文がリレーショナルデータベースに記録されたことを購入者に伝える。

```plantuml
@startsalt
{
  {* EC サイト最小購入フロー | 注文完了 }
  ..
  { 注文を受け付けました。 }
  { 注文番号: ORD-20260703-0001（表示例） }
  ..
  { [ 商品一覧へ戻る ] }
}
@endsalt
```

注文の識別子の形式は例示であり、確定は Construction の Functional Design が扱う。
