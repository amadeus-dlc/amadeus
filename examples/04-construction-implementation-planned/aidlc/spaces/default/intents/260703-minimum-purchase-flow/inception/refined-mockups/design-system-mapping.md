# Design System Mapping：260703-minimum-purchase-flow

このプロジェクトに確立済みのデザインシステムはない（greenfield、単独開発者）。
そのため、画面要素は汎用 UI 部品の語彙で対応づける。
具体の UI コンポーネントライブラリの選定は、Construction の NFR Requirements（技術スタック判断）で扱う。

## 対応

| 画面要素 | デザインシステムの部品 | 備考 |
|---|---|---|
| 商品一覧 | テーブル | 商品名、価格、在庫、数量、操作の 5 列 |
| 数量入力 | 数値入力フィールド | 既定値 1、最小値 1、整数のみ（GD004） |
| 選択ボタン | プライマリボタン | 在庫なし、在庫不明の場合は無効状態 |
| 在庫状況 | テキストラベル | あり、なし、不明の 3 状態 |
| エラー表示 | アラートバナー | 在庫参照失敗（R008）、在庫不足（R007）で使用 |
| 再試行ボタン | セカンダリボタン | アラートバナーに併置 |
| 注文内容 | テーブル | 商品名、数量、金額の 3 列と合計行 |
| 注文を作成するボタン | プライマリボタン | 注文内容確認画面の確定操作 |
| 商品一覧へ戻るボタン | セカンダリボタン | 全画面共通の戻り導線 |
| 注文番号 | テキスト表示 | 注文完了画面で提示 |
