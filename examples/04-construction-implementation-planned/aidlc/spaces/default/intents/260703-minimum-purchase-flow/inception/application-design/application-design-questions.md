# Questions：Application Design

| # | 確認したいこと | 推奨回答 | 回答 |
|---|---|---|---|
| 1 | 新しいコンポーネントの責務境界はどこか | UI、サービス層、ドメインの 3 層 | UI の 3 画面、サービス層の 2 サービス、ドメインの商品カタログ、在庫参照、注文に分けた（GD005） |
| 2 | 既存アーキテクチャとの整合で譲れない点は何か | constraint-register の 4 制約 | TypeScript と Node.js、購入者向け Web UI、在庫管理システムとの REST API 連携、注文のリレーショナルデータベース記録の 4 つを設計の前提にした |
| 3 | サービスの分割単位はどうするか | 購入フローの局面ごとに 2 分割 | 商品一覧サービスと注文作成サービスの 2 つ。要求の境界と一致させた（GD007） |
