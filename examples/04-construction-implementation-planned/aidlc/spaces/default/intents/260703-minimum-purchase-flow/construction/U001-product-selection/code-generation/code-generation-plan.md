# Code Generation Plan：商品選択

## 変更対象

| ファイル | 変更内容 |
|---|---|
| 未確認：商品一覧画面 | 商品名と価格の一覧を表示し、購入者が 1 つの商品を選択できる UI を実装する。 |
| 未確認：商品行 | 1 商品の表示と選択操作をまとめ、商品識別子、商品名、価格を選択済み商品として渡せるようにする。 |
| 未確認：商品一覧提供 | 注文対象になる商品を商品識別子、商品名、価格として返す処理を実装する。取得元は未確認のため、B001 では最小の取得方式を採用する。 |
| 未確認：商品サービス | 商品一覧提供から商品名と価格を取得し、商品一覧画面へ渡す経路を実装する。B001 では在庫参照の結果を選択可否に使わない。 |
| 未確認：商品選択のテスト | 商品名と価格の表示、1 商品の選択、選択済み商品の受け渡しを検証する。 |

## 変更順序

1. 商品識別子、商品名、価格を持つ商品と選択済み商品の型またはデータ構造を定義する。
2. 商品一覧提供を実装し、注文対象の商品を取得できる最小の経路を作る。
3. 商品サービスで商品一覧提供を呼び出し、商品一覧画面が扱う表示用の一覧へ変換する。
4. 商品一覧画面と商品行を実装し、商品名、価格、選択操作を表示する。
5. 商品選択操作で、選択済み商品の商品識別子、商品名、価格を注文内容確認画面へ渡す。
6. 商品一覧の表示と選択済み商品の受け渡しをテストする。

## 検証方法

- 商品一覧画面を開くと、商品名と価格が表示されることを確認する。
- 商品を 1 つ選択すると、商品識別子、商品名、価格を持つ選択済み商品が注文内容確認画面へ渡ることを確認する。
- B001 では在庫状況の表示と選択可否を検証対象に含めないことを確認する。
- 商品一覧の取得元、商品識別子の形式、商品が 0 件の場合の表示は `未確認` として残す。
