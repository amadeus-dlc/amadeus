# Code Generation Plan：注文作成

## 変更対象

| ファイル | 変更内容 |
|---|---|
| 未確認：注文内容確認画面 | 選択済み商品の商品名、数量、金額、合計金額を表示し、数量指定と注文作成操作を提供する。 |
| 未確認：数量入力 | 1 以上の整数だけを受け付け、数量変更に応じて合計金額を更新する。 |
| 未確認：注文完了画面 | 作成された注文の識別子を表示する。 |
| 未確認：注文作成 | 選択済み商品、数量、注文可否から注文記録候補を組み立て、数量と合計金額を検証する。 |
| 未確認：注文記録 | 注文の識別子、商品、数量、金額、合計金額、作成日時をリレーショナルデータベースへ記録する。 |
| 未確認：注文サービス | 注文作成と注文記録を呼び出し、注文識別子を注文完了画面へ返す。 |
| 未確認：注文作成のテスト | 数量指定、合計金額算出、注文記録、注文識別子表示を検証する。 |

## 変更順序

1. 注文内容、注文、注文可否、注文記録候補の型またはデータ構造を定義する。
2. 数量が 1 以上の整数であること、合計金額が価格と数量から算出されることを注文作成で検証する。
3. 注文記録を実装し、注文の識別子、商品、数量、金額、合計金額、作成日時をリレーショナルデータベースへ保存する。
4. 注文サービスで注文作成と注文記録を接続し、注文識別子を返す。
5. 注文内容確認画面と数量入力を実装し、選択済み商品の商品名、数量、金額、合計金額を表示する。
6. 注文作成操作から注文サービスを呼び出し、注文完了画面へ注文識別子を渡す。
7. 注文完了画面を実装し、注文識別子を表示する。
8. 数量指定、合計金額、注文記録、注文識別子表示をテストする。

## 検証方法

- 注文内容確認画面に、選択済み商品の商品名、数量、金額、合計金額が表示されることを確認する。
- 数量を 1 以上の整数へ変更すると、合計金額が数量に応じて更新されることを確認する。
- 0 以下の数量と整数以外の数量では注文を作成しないことを確認する。
- 注文作成操作で、注文の識別子、商品、数量、金額、合計金額、作成日時がリレーショナルデータベースへ記録されることを確認する。
- 注文完了画面に注文識別子が表示されることを確認する。
- B001 では在庫管理システムとの REST API 連携を検証対象に含めず、注文可否の実入力元は B002 で接続する前提として確認する。
- 注文識別子の形式、注文記録の参照方法、リレーショナルデータベース製品名、注文記録失敗時の表示は `未確認` として残す。
