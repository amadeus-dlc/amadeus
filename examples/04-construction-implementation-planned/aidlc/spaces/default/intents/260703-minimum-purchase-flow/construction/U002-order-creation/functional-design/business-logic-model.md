# Business Logic Model：注文作成

## 目的

購入者が選択済み商品に数量を指定し、注文をリレーショナルデータベースへ記録できる状態を作る。

注文作成には新しいデータモデルと、在庫参照の結果に基づく注文可否の業務ルールがある。
そのため Functional Design を実行し、Code Generation 前に注文の入力、出力、記録内容、注文可否を定義する。

## 対象 Unit

U002 注文作成

## 業務ロジック

注文内容確認画面は、選択済み商品の商品名、数量、金額、合計金額を表示する。
数量は 1 以上の整数であり、既定値は 1 とする。

注文作成は、選択済み商品、数量、注文可否を入力として受け取り、注文を組み立てる。
注文可否が注文可能であり、数量が妥当な場合だけ注文記録へ渡す。

注文記録は、注文の識別子、商品、数量、金額、合計金額、作成日時をリレーショナルデータベースへ記録する。
注文完了画面は、発行された注文の識別子を表示する。

B001 では在庫管理システムとの連携を行わない。
ただし、注文作成の契約には注文可否を含める。
B002 で在庫参照結果が統合された後は、在庫ありの場合だけ注文可能になり、在庫なしまたは確認不可の場合は注文不可になる。

## 入力

- 選択済み商品
- 数量
- 注文可否
- 注文作成操作

## 出力

- 注文内容
- 注文記録
- 注文の識別子

## 未確認事項

- 注文識別子の形式は未確認である。
- 注文記録の参照方法は未確認である。
- B001 の Code Generation で使うリレーショナルデータベース製品は、この指示では確定済みとして扱うが、具体名は未確認である。
