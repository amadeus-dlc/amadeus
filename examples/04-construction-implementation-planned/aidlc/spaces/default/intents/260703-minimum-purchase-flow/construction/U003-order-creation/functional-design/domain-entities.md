# Domain Entities：注文作成

## 目的

注文作成が扱うドメインの概念と関係を定義する。

## Domain Entity

| Entity | 責務 | 主な属性 |
|---|---|---|
| 注文 | 購入者の注文を表す。注文番号の採番と記録内容の整合を守る | 注文番号、商品識別子、商品名、価格、数量、金額、合計金額、作成日時 |
| 注文番号 | 記録済みの注文を一意に識別する値 | 形式「ORD-YYYYMMDD-NNNN」（作成日と日内連番 4 桁） |

## 関係

注文は商品カタログの商品を商品識別子で参照するが、商品名と価格は注文作成時点の値を注文自身が保持する。
注文の作成可否は、在庫参照（U001）が返す在庫状況に依存する。

本 Intent の注文対象は 1 商品である。
要求（R003、R005）が金額と合計金額を区別しているため、属性としては両方を保持し、複数商品の注文は将来の拡張余地として残す。

## Domain Map と Context Map への反映候補

| 候補 | 種別 | 判断 |
|---|---|---|
| 注文作成と在庫参照の間の語彙（在庫状況、参照失敗） | 共有境界 | 見送り |

見送りの理由は次である。
注文作成と在庫参照は、どちらも境界づけられたコンテキスト BC001 購入フローの内側の領域であり、この語彙の共有はコンテキスト境界の変更ではない。

## 未確認事項

- 注文の保持期間と削除の要否は未確認である。本 Intent は注文の作成と記録までを対象にする。
