# Domain Entities：在庫参照

## 目的

在庫参照が扱うドメインの概念と関係を定義する。

## Domain Entity

| Entity | 責務 | 主な属性 |
|---|---|---|
| 在庫状況 | 商品ごとの在庫の有無と数量を表す | 商品識別子、在庫の有無、在庫数量 |
| 参照失敗 | 在庫状況が得られなかったという事実を表す | なし（事実だけを持ち、失敗の理由を区別しない） |

## 関係

在庫状況は、商品識別子で商品カタログの商品と対応する。
在庫参照の結果は、在庫状況の一覧または参照失敗のどちらか一方である。

## Domain Map と Context Map への反映候補

| 候補 | 種別 | 判断 |
|---|---|---|
| 購入フロー（本 Intent のアプリケーション）と在庫管理システム（EXT001）を別の境界づけられたコンテキストとして扱う | 共有境界 | 採用 |
| 購入フローから在庫管理システムへのコンテキスト間依存（組織パターンは順応者、統合パターンは腐敗防止層（ACL）） | コンテキスト間依存 | 採用 |

採用の根拠は次である。
在庫管理システムは EXT001 として確定済みの外部システムであり、購入フロー側から API 仕様に影響を与えられないため、組織パターンは順応者にする。
在庫参照が REST API の応答をドメインの語彙（在庫状況、参照失敗）へ変換し、失敗の検出を集約するため（GD006）、統合パターンは腐敗防止層（ACL）にする。

## 未確認事項

- 在庫数量の粒度（数量が得られるか、在庫の有無だけか）は、API 仕様とあわせて未確認である。数量が得られない場合は、在庫不足の判定方法（U003）を見直す。
