# Business Logic Model：注文作成

## 目的

数量指定と注文内容の確認から、注文のリレーショナルデータベースへの記録と注文の識別子の表示までを一連で行うロジックを定義する。

## 対象 Unit

U003 注文作成。
注文内容確認画面、注文完了画面と注文サービス（注文作成、注文記録）で、数量指定から注文のリレーショナルデータベース記録と注文の識別子表示までを成立させる（R003、R004）。

## 業務ロジック

注文内容の確認では、商品選択（U001）から引き継いだ商品識別子に商品名と価格を対応づけ、数量（既定値 1）を受け付け、金額（価格 × 数量）と合計金額を算出して表示する（R003）。
数量を変更すると、金額と合計金額を数量に応じて更新する。
0 以下の数量と整数以外の数量は受け付けず、数量は 1 以上の整数で指定する旨を表示して「注文を作成する」を無効にする（interaction-spec.md の数量の指定）。

注文の作成では、注文作成コンポーネントが数量が 1 以上の整数であることを検証し、商品の価格から金額と合計金額を算出して注文を組み立て、注文記録へ渡す（R003、R004、component-methods.md の注文作成）。
注文記録は、注文をリレーショナルデータベースに記録し、保存時に注文の識別子を発行する（R004、application-design の decisions.md の 4）。
注文完了画面は、発行された注文の識別子を表示する（R004）。

注文の記録に失敗した場合は、注文が作成されなかった旨を表示して注文内容確認画面に留まり、注文の識別子を発行しない（interaction-spec.md の注文の作成、decisions.md の 4）。
認証と決済は行わず、注文作成は決済なしで完了する（R004）。

注文可否は在庫参照の結果に基づき、注文作成時の在庫の再確認は行わない（decisions.md の 3）。
在庫がない商品と在庫を確認できない商品は、商品一覧画面の選択段階で止まるため、注文作成は選択できた商品だけを前提にできる（R002、R006、unit-of-work-dependency.md）。
B001 の時点では在庫参照（U002）が未統合であり、この選択段階の判定は B002 の統合後に効く。

## 入力

- 商品識別子（商品選択から引き継ぐ）、数量（既定値 1、1 以上の整数）

## 出力

- 注文の識別子（注文を記録した場合）
- または数量の検証エラー、記録の失敗（注文を記録しなかった場合）
