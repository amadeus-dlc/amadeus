# Construction Design

## 概要

- 対象 Bolt は B001 注文作成である。
- B001 は、注文内容、購入者情報、販売可能在庫の参照結果をもとに、BC004 販売管理の注文を作成する。
- R004 と UC003 を主対象にし、R002 と R003 は注文作成の入力前提として扱う。
- 決済詳細、売上確定、在庫引当、出荷は作成処理に含めない。
- Design Gate の evidence は、このファイルを指す。

## Domain Design

- **注文**：顧客が確認済みの注文内容をもとに作成される販売管理の記録である。
- **注文内容**：選択された商品と購入数量を含む、顧客が注文作成前に確認した内容である。
- **購入者情報**：注文へ記録する顧客由来の情報である。
  項目は未確認であるため、Construction 実装では既存入力または最小入力に合わせて扱う。
- **販売可能在庫の参照結果**：注文作成前に確認された販売可能在庫の有無または数量を表す結果である。
  在庫引当は含めない。
- 注文作成の事前条件は、注文内容、購入者情報、販売可能在庫の参照結果が注文作成入力としてそろっていることである。
- 注文作成の事後条件は、作成された注文に注文内容と購入者情報が記録されることである。
- 不変条件は、注文作成時に決済、売上確定、在庫引当、出荷を発生させないことである。

## Logical Design

- 注文作成入口は、UC003 の「顧客が確認済みの注文内容で注文作成を依頼する」相互作用を受け取る。
- 注文作成制御は、注文内容、購入者情報、販売可能在庫の参照結果を検査し、注文作成に必要な内容が不足していないことを確認する。
- 販売可能在庫の参照結果は、注文作成前に参照済みの結果として受け取る。
  B001 では在庫管理へ問い合わせず、結果の有無と注文作成可否だけを扱う。
- 注文エンティティは、注文内容、購入者情報、販売可能在庫の参照結果、作成結果として顧客へ示す識別情報候補を保持する。
- 注文番号、注文状態、永続化方式は未確認である。
  Construction 実装では既存コードがある場合は既存方式へ合わせ、ない場合は Task 内で最小の作成結果として扱う。
- 既存コード調査では、この workspace にアプリケーション実装コードは確認できなかった。
  そのため、実装対象の配置は Implementation Execution で確定する。

## 実装設計

- B001/T001 では、注文作成入力として注文内容、購入者情報、販売可能在庫の参照結果を表す構造を定義する。
- B001/T002 では、注文作成に必要な内容の不足を検出し、注文作成へ進めない条件を明示する。
- B001/T003 では、注文内容と購入者情報を持つ注文を作成し、作成結果を返す処理を実装する。
- B001/T004 では、決済、売上確定、在庫引当、出荷を呼び出さない境界を実装上の依存関係として固定する。
- B001/T005 では、UC003 の基本フローに沿って、作成された注文を顧客へ示す出力を整える。
- 実装ファイルの具体的な配置は未確認である。
  Implementation Execution では、既存のソース配置がある場合は既存構造へ合わせ、ない場合は最小の販売管理モジュールとして追加する。
