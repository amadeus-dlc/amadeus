# D004: Bolt 分割判断

## 背景

- Bolt 候補は、B001 注文作成、B002 注文内容確認、B003 商品選択である。
- Construction では Task を Bolt ごとに分解する。

## 判断

- 採用。
- B001 注文作成、B002 注文内容確認、B003 商品選択に分ける。

## 理由

- 商品選択、注文内容確認、注文作成は実施順序と完了条件が異なるため。
- B003 から B002、B002 から B001 へ依存させることで、Construction の作業順序を明確にできるため。
- Bolt は Unit Design Brief を参照し、Task や実装設計は先取りしないため。

## 影響

- B001 は U002 注文作成を対象にする。
- B002 と B003 は U001 最小購入フローを対象にする。
- Task は Construction phase で生成する。
