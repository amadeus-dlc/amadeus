# D002: Unit と Bolt の分割

## 背景

- Unit 候補として、最小購入フローと注文作成が示された。
- Bolt 候補として、`B001 注文作成`、`B002 注文内容確認`、`B003 商品選択` が示された。

## 判断

- `U001 最小購入フロー` は `R001`、`R002`、`R003` を扱う。
- `U002 注文作成` は `R004` を扱う。
- `B003 商品選択` は `R001` を扱う。
- `B002 注文内容確認` は `R001`、`R002`、`R003` を扱う。
- `B001 注文作成` は `R004` を扱う。

## 理由

- 注文作成前の確認と、注文そのものの作成は価値境界が異なるため。
- `R004 注文作成` は `B001` の責務とする指定があるため。
- `B002` に `R004` を含めると、注文内容確認と注文作成の責務が混ざるため。

## 影響

- traceability では、`R004` の Bolt を `B001` のみにする。
- traceability では、`B002` の要求に `R004` を含めない。
