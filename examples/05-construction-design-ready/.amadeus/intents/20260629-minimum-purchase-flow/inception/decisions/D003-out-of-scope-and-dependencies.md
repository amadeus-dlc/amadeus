# D003: 対象外範囲と依存範囲

## 背景

- Ideation では、会員登録、ログイン、顧客台帳、購入履歴管理、決済詳細、売上確定、在庫引当、入荷、棚卸し、出荷が対象外として整理された。
- 販売可能在庫の確認では、在庫管理への依存が必要になる。

## 判断

- 在庫管理を Intent 内で明示する場合は、`SD006 在庫管理` として扱う。
- `BC001 販売管理` は販売可能在庫の参照で在庫管理に依存する。
- 在庫引当は今回の範囲外にする。
- 購入者情報は注文に記録するが、会員登録、ログイン、顧客台帳、購入履歴管理は今回の範囲外にする。

## 理由

- 既存の steering layer のサブドメイン ID と衝突させないため。
- 最小購入フローでは、注文作成前の販売可否確認までが必要であり、在庫引当や顧客台帳の詳細管理は後続領域で扱うため。

## 影響

- `domain/subdomains.md` では、`SD001` から `SD005` の既存 ID を維持し、在庫管理を `SD006` として追加する。
- `domain/bounded-contexts.md` では、Intent が所有する境界づけられたコンテキストを `BC001 販売管理` だけにする。
