# Upstream AI-DLC sync plan: v2.2.0 to v2.3.0

## Executive summary

This plan compares upstream `awslabs/aidlc-workflows` v2.2.0 to v2.3.0 and maps
every change onto Amadeus. The window contains 19 patch releases (2.2.1–2.2.19)
plus the 2.3.0 plugin mechanism. The headline item is the **AIDLC plugin
mechanism** (new `plugins/<name>/` authoring surface, packager plugin
projections, a compose hook, and stage-schema extensions), which is entirely
missing in Amadeus. The patch window contributes engine correctness fixes
(bolt_dag self-heal, gate-revision backstop, autonomous swarm batch advance),
engine features (unit kinds, unit-major iteration, scope cost preview),
workspace-detection improvements (nested roots, git submodules), harness
integration fixes (`process.execPath` spawn, Kiro IDE hook context,
`$CLAUDE_PROJECT_DIR` quoting), and reviewer-quality changes (shell-sourced
review dates, bounded reviewer read scope). Several items are already
equivalent in Amadeus (`optional_produces:`, agent `model:` frontmatter,
per-intent learnings paths) and are classified SKIP with evidence.

Per the user's instruction, the follow-up Intent will be run **through the
ideation phase only and then parked**; construction of the adopted items is a
later, separately approved continuation.

## Residual whole-tree drift

Beyond the tag window, Amadeus is a deliberately diverged fork; a byte-level
whole-tree comparison is not meaningful. Structural residual drift, from the
pinned target tree (`git ls-tree` of `29a31f78…`) versus Amadeus `a326f47bc`:

- Namespace: `aidlc*` → `amadeus*` everywhere (tools, hooks, markers, dirs).
- Layout: upstream `core/` and `harness/<name>/` map to
  `packages/framework/core/` and `packages/framework/harness/<name>/`;
  upstream has no `packages/` split.
- Harness set: upstream ships 4 projections (claude, codex, kiro, kiro-ide);
  Amadeus ships 6 (adds cursor, opencode). Every adopted harness-wide change
  must additionally cover the two Amadeus-only harnesses.
- Amadeus-only surfaces with no upstream counterpart: `packages/setup`,
  `scripts/` release/dev tooling, the election CLI, session skills,
  `amadeus/` workspace tree, coverage ratchet/registry machinery.
- Upstream-only surfaces not yet in Amadeus: `plugins/`, `dist/plugins/`,
  `scripts/plugin-hooks-template`, `roadmap.html`, `tests/tsconfig.json`
  (Amadeus uses `tsconfig.tests.json` at the repo root instead — equivalent).
- Pre-2.2.0 upstream behaviors are outside this window and are NOT certified
  equivalent by this plan (baseline is REVIEWED, not APPLIED).
