# Design Agent

> **エージェント詳細解説** · [ユーザーガイド](../00-introduction.ja.md) › [エージェント](../06-agents.ja.md) › [詳細解説](README.ja.md) · 技術リファレンス: [design-agent](../../reference/agents/design-agent.ja.md)

amadeus-design-agent は UX/UI デザイナーです。Ideation ではワイヤーフレームとコンセプトスケッチを生成し、Inception ではそれらをインタラクション仕様を伴う高忠実度のモックアップへと発展させます。情報アーキテクチャ、ナビゲーション設計、レスポンシブな挙動、アクセシビリティ要件を定義します。UI を持たないプロジェクトでは、システムコンテキスト図と API 体験設計を生成します。

amadeus-design-agent は 2 つのステージをリードし、他の 2 つをサポートします。ユーザーに面するすべてのインターフェースが、使いやすく、アクセシブルで、デザインシステムの標準に一貫していることを保証します。

## リードするステージ

| ステージ | フェーズ | 説明 |
|-------|-------|-------------|
| 1.6 Rough Mockups | Ideation | 低忠実度のワイヤーフレームとコンセプトの可視化 |
| 2.5 Refined Mockups | Inception | インタラクション仕様とアクセシビリティを伴う高忠実度のモックアップ |

## サポートするステージ

| ステージ | フェーズ | 貢献 |
|-------|-------|-------------|
| 2.4 User Stories | Inception | インタラクションの詳細と UX 受け入れ基準でストーリーを充実させる |
| 2.6 Application Design | Inception | UI コンポーネント仕様を提供 |

## 期待できること

amadeus-design-agent がアクティブなとき、画面レイアウト、ユーザーフロー、インタラクションパターンの詳細な記述を markdown で生成します。対象デバイス、アクセシビリティ要件、デザインシステムの選好について問いかけます。インタラクションを具体的な画面状態と遷移 — ローディング、成功、エラー、空、部分的な状態 — の観点で記述します。

## 協働の仕方

amadeus-design-agent は amadeus-product-agent から intent とユーザーストーリーを、amadeus-architect-agent からコンポーネントの制約を受け取ります。そのモックアップとインタラクション仕様は、実装のために amadeus-developer-agent へ、UX 受け入れテストのために amadeus-quality-agent へ引き渡されます。

## 主要な原則

- スキャナビリティのために設計する — 重要なアクションは即座に見えなければならない
- 一貫性は認知負荷を減らす — すべてのインタラクションパターンは予測可能であるべき
- エラーメッセージよりエラー防止 — エラーを起こしにくくする
- WCAG アクセシビリティ準拠はベースラインであり、努力目標ではない
- 最悪のケースのために設計する — 空の状態、エラー状態、遅い接続
