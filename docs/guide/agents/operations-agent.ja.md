# Operations Agent

> **エージェント詳細解説** · [ユーザーガイド](../00-introduction.md) › [エージェント](../06-agents.md) › [詳細解説](README.md) · 技術リファレンス: [operations-agent](../../reference/agents/operations-agent.md)

amadeus-operations-agent はサイトリライアビリティエンジニアかつインシデントマネージャーです。デプロイされたシステムが可観測で、レジリエントで、継続的に改善されることを保証します。CloudWatch ダッシュボードとアラームから、X-Ray トレーシング、SLO トラッキング、インシデント対応ランブック、カオスエンジニアリング検証に至るまで、運用レイヤーを所有します。決定的に重要なのは、本番環境のインサイトを次のイテレーションのために Ideation へ還流させることで、フィードバックループを閉じる点です。

amadeus-operations-agent は Operation フェーズで 3 つのステージをリードします。監視のセットアップコマンド、ランブックスクリプト、診断ツールを実行するための Bash アクセスを持ちます。

## リードするステージ

| ステージ | フェーズ | 説明 |
|-------|-------|-------------|
| 4.4 Observability Setup | Operation | ダッシュボード、アラーム、トレーシング、構造化ロギング、カスタムメトリクス |
| 4.5 Incident Response | Operation | SSM ランブック、インシデント計画、エスカレーションマトリクス、オンコール体制 |
| 4.7 Feedback & Optimization | Operation | SLO レポート、コスト分析、ドリフト検出、フィードバックループ |

amadeus-quality-agent は 4.6 Performance Validation をリードします。このエージェントが 4.4 でセットアップする運用テレメトリとベースラインは、その作業を非公式に支えますが、4.6 の正式なサポートエージェントではありません。

## サポートするステージ

なし。amadeus-operations-agent はリード専任のエージェントであり — 他のどのエージェントのステージもサポートしません。そのリードはすべて Operation にあり、そこでライフサイクルのループを閉じます。

## 期待できること

amadeus-operations-agent がアクティブなとき、監視の選好、SLO 目標、インシデント対応プロセス、オンコール体制について問いかけます。CloudWatch ダッシュボード構成、閾値と通知先を伴うアラーム定義、X-Ray トレーシングのセットアップ、一般的なシナリオ(サービス再起動、キャッシュフラッシュ、フェイルオーバー)向けの SSM ランブック、エスカレーションパスを伴うインシデント重大度の定義を生成します。

Feedback & Optimization(最終ステージ)の間、本番環境のメトリクスを分析し、最適化の機会を特定し、運用上のインサイトを次の開発サイクルのために amadeus-product-agent へ還流させるフィードバックループドキュメントを生成します。

## 協働の仕方

amadeus-operations-agent は amadeus-aws-platform-agent からプロビジョニングされたインフラを、amadeus-pipeline-deploy-agent からデプロイされたサービスを受け取ります。パフォーマンスのベースラインと SLO 検証については amadeus-quality-agent と、アプリケーションレベルのロギング改善については amadeus-developer-agent と協働します。そのフィードバックレポートは Operation から Ideation への橋渡しであり、ライフサイクルのループを完成させます。

## 主要な原則

- 包括的なテレメトリを収集するが、アラートはユーザーに影響する問題に限る — アラート疲れは対応を劣化させる
- SLO が信頼性の目標を定義する。それ以外のすべてはそこから導かれる
- すべてのインシデントは学習の機会である — 非難なきポストモーテムがインシデントを改善へと変える
- テストされていないレジリエンス機構は前提にすぎない — カオスエンジニアリングがそれを検証する
- Ideation へ還流しない本番環境のインサイトは、無駄になった学習である
- 手作業の運用トイルは排除しなければならない — 反復可能なランブックのステップはすべて自動化する
