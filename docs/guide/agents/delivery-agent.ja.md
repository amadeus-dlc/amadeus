# Delivery Agent

> **エージェント詳細解説** · [ユーザーガイド](../00-introduction.ja.md) › [エージェント](../06-agents.ja.md) › [詳細解説](README.ja.md) · 技術リファレンス: [delivery-agent](../../reference/agents/delivery-agent.ja.md)

amadeus-delivery-agent はエンジニアリングマネージャーかつデリバリープランナーです。スコープ定義とアーキテクチャ設計を、チーム割り当て、ビルド順序、シーケンスを伴う実行可能なデリバリー計画へと翻訳します。Ideation から Construction への橋渡しとなるイニシアチブブリーフを所有し、完全なトレーサビリティを備えたスムーズなフェーズ引き継ぎを保証します。

amadeus-delivery-agent は Ideation と Inception にまたがる 3 つのステージをリードします。設計されたものが実際にデリバリー可能か、そしてどの順序でかを保証する責任を負うエージェントです。

## リードするステージ

| ステージ | フェーズ | 説明 |
|-------|-------|-------------|
| 1.5 Team Formation | Ideation | 必要なスキルを評価しチーム構造を構成する |
| 1.7 Approval & Handoff | Ideation | フェーズゲート承認のためのイニシアチブブリーフをまとめる |
| 2.8 Delivery Planning | Inception | Bolt シーケンス(2.7 の DAG を通じた経済的な順序付け)を計画し、チーム配分、リスク/シーケンスの根拠、外部依存関係マップを伴う |

## サポートするステージ

| ステージ | フェーズ | 貢献 |
|-------|-------|-------------|
| 1.4 Scope Definition | Ideation | デリバリーの実現可能性に照らしてスコープを検証 |
| 2.7 Units Generation | Inception | ユニットの粒度を計画上のニーズに合わせる |

## 期待できること

amadeus-delivery-agent がアクティブなとき、シーケンスと実現可能性に焦点を当てます。チームの規模、利用可能な専門知識、デリバリーの選好について問いかけます。作業単位をビルド順序へマッピングし、クリティカルパスを特定し、進行をブロックしうる依存関係にフラグを立てる構造化された計画を生成します。

## 協働の仕方

amadeus-delivery-agent は amadeus-product-agent からスコープと優先順位を、amadeus-architect-agent からユニット仕様と複雑度の見積もりを受け取ります。そのデリバリー計画は、ビルド順序とチーム割り当てを理解するためにすべての Construction エージェントによって消費されます。

## 主要な原則

- 計画は生きたドキュメントである — 変更できない計画は失敗する
- 小さなバッチ、素早いフィードバック — より小さなインクリメントほど早期にリスクを表面化させる
- すべての作業単位は要件まで遡れなければならない
- フェーズ移行には明示的な完全性チェックが必要である
- 自信は Bolt を出荷するごとに獲得される — 出荷された各 Bolt が次の Bolt のリスクを下げる
